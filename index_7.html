Функція map() — масштабування діапазонів

Функція map() переносить значення з одного діапазону в інший пропорційно. Оперує лише цілими числами — дробова частина при пропорційному переносі відкидається (не округлюється).

Синтаксис та параметри

map(value, fromLow, fromHigh, toLow, toHigh)
// value — значення для перетворення
// fromLow/fromHigh — поточний діапазон (вхід)
// toLow/toHigh — новий діапазон (вихід)
// повертає long (ціле!)
📐 Приклад 1: analogRead → analogWrite (ШІМ)+

АЦП дає 0–1023, ШІМ приймає 0–255. Пряме масштабування:

analogRead → map → analogWrite
void setup() { pinMode(A1, INPUT); pinMode(10, OUTPUT); } void loop() { int value = analogRead(A1); // 0–1023 value = map(value, 0, 1023, 0, 255); // → 0–255 analogWrite(10, value); // ШІМ на LED }
🔄 Приклад 2: Інверсія діапазону+

map() може інвертувати діапазон, якщо toLow > toHigh:

Інвертований map()
// Чим більше value → тим менший результат value = map(val, 0, 1023, 255, 0); // val=0 → 255 (максимум) // val=512 → 127 // val=1023→ 0 (мінімум)
Інверсія корисна для: «чим далі рука від датчика — тим яскравіший LED», або «чим вологіше — тим тихіше сигнал».
⚠ Особливості та обмеження map()+
  • Тільки цілі числа — дробова частина відкидається. map(2, 0, 4, 0, 1) поверне 0, а не 0.5!
  • Не затискає до меж — якщо value виходить за fromLow..fromHigh, результат теж вийде за межі toLow..toHigh. Використовуйте constrain() для захисту.
  • Зворотній зв'язок: map(map(x,a,b,c,d),c,d,a,b) = x (симетрія)
map() + constrain()
value = map(sensorVal, 0, 1023, 0, 255); value = constrain(value, 0, 255); // гарантований діапазон
Псевдовипадкові числа — random() та randomSeed()

Комп'ютер не може генерувати справді випадкові числа — лише псевдовипадкові: детерміновані послідовності з виглядом хаотичних. Генератор стартує з «зерна» (seed) і відтворює ту саму послідовність при кожному запуску.

Ключові функції

ФункціяОписПриклад
random(max)Число від 0 до max-1random(10) → 0..9
random(min,max)Число від min до max-1random(100,200) → 100..199
randomSeed(seed)Ініціалізація генератораrandomSeed(analogRead(A1))
🎲 Тип byte: random для цілих чисел+
random() · тип byte
byte data; void setup() { Serial.begin(9600); } void loop() { data = random(100, 200); // 100..199 Serial.println(data); delay(250); }
🎲 Float з випадковими знаками після коми+

МК генерує лише цілі. Для float генеруємо окремо цілу та дробову частини:

random() · float з комою
float data, data1, data2; void loop() { data1 = random(0, 10); // цілі: 0..9 data2 = random(0, 100); // після коми: 0..99 data = data1 + data2 / 100.0; // напр.: 5.73 Serial.println(data); delay(250); }
🎲 randomSeed() — щоразу різна послідовність+

Без randomSeed() — кожен перезапуск дає однакову послідовність. З analogRead незаземленого піна — шум дає щоразу різне зерно:

randomSeed(analogRead(A1))
void setup() { Serial.begin(9600); // A1 нічого не підключено — «шум» як зерно randomSeed(analogRead(A1)); } void loop() { Serial.println(random(100, 200)); delay(250); }
Вільний аналоговий пін «ловить» електромагнітний шум від Wi-Fi, стільникових телефонів, мережі 50 Гц — це забезпечує різне зерно при кожному старті.
PIR Датчик руху HC-SR501

PIR (Passive Infrared) — пасивний інфрачервоний датчик руху. Реєструє зміну теплового ІЧ-поля у зоні огляду — рух теплих тіл (людини, тварини). «Пасивний» — тому що не випромінює, лише приймає.

Живлення5В – 20В
Вихідний сигналDigital HIGH (3.3В) при русі
Кут огляду120°
Дальністьдо 7 м (регулюється)
Затримка спрацювання0.3 – 18 с (підстроювач)

Після подачі живлення PIR потребує 30–60 секунд прогріву для стабілізації. У цей час можливі хибні спрацювання — це нормально.
Геркон (Reed Switch) — магнітокерований контакт

Геркон — герметизований магнітокерований контакт. Скляна колба з двома феромагнітними контактами, запаяна в захисному газі. При піднесенні магніту контакти замикаються (NO-тип) або розмикаються (NC-тип).

Два варіанти підключення

ВаріантСхемаpinModeАктивний станПереваги
Варіант 1 (зовн. R)5V→Геркон→D2 + 10кОм до GNDINPUTHIGH (магніт є)Наочний
Варіант 2 (PULLUP)Геркон→D2→GNDINPUT_PULLUPLOW (магніт є)Без зовн. R, надійний
⚡ Чому INPUT_PULLUP кращий за INPUT + 10кОм?+
  • Менше деталей — резистор 10 кОм вбудований у МК (≈20 кОм). Схема спрощується.
  • Стабільність — пін не «плаває» у невизначеному стані між HIGH і LOW.
  • Безпека — якщо дріт геркона торкнеться заземленого корпусу — не буде КЗ живлення.
  • Логіка інвертована — активний стан LOW (магніт є). Умова: if(reedState == LOW)
Застосування: охорона дверей/вікон (магніт на рамі, геркон на дверях), лічильники обертів, кнопки у герметичних пристроях, електронні замки, сигналізації.

Лістинг 5-2 · PIR → LED автоматика
#define PIN_PIR 2 // PIR OUT → D2 #define PIN_LED 13 // LED → D13 void setup() { Serial.begin(9600); pinMode(PIN_PIR, INPUT); // читаємо з PIR pinMode(PIN_LED, OUTPUT); // керуємо LED } void loop() { int pirVal = digitalRead(PIN_PIR); Serial.println(pirVal); if (pirVal) { // HIGH = рух виявлено digitalWrite(PIN_LED, HIGH); Serial.println("Виявлено рух"); delay(2000); // тримаємо LED 2 сек } else { digitalWrite(PIN_LED, LOW); } }
Додайте лічильник спрацювань у глобальну змінну та виводьте в Serial Monitor повідомлення «Спрацювань: N».

Project 5-4 · DHT11 · 1-wire · Serial Monitor
Вимірювання температури та вологості

Цифровий давач DHT11 вимірює температуру і вологість через 1-wire протокол з одним сигнальним дротом. Використовуємо бібліотеку DHT від Adafruit. Дані виводяться в Serial Monitor.

Необхідне обладнання
Arduino UNO
Давач DHT11
Резистор 10 кОм (pull-up)
Макетна плата
З'єднувальні дроти
Чому pull-up резистор 10 кОм? Лінія Data використовує протокол 1-wire з «відкритим стоком». Без підтяжки до VCC сигнал «плаває» між HIGH і LOW → NaN у зчитуванні. Вбудований pullup Arduino (~100 кОм) занадто слабкий для DHT11.
// Принципова схема · DHT11 з pull-up 10кОм
  • 1
    Підключіть DHT11: VCC→5V, DATA→D2, GND→GND. Між DATA та VCC підключіть резистор 10 кОм (pull-up).
  • 2
    Встановіть бібліотеку DHT sensor library (Adafruit) через менеджер бібліотек IDE. Разом із нею встановиться Adafruit Unified Sensor.
  • 3
    Введіть Лістинг 5-4. Зверніть увагу: isnan(t) || isnan(h) — перевірка на «Not a Number», якщо датчик не відповів.
  • 4
    Завантажте. Відкрийте Serial Monitor (9600 бод). Ви маєте побачити: Humidity: 55.00 % Temperature: 23.00 *C
Лістинг 5-4 · DHT11 → Serial Monitor
#include "DHT.h" #define DHTPIN 2 #define DHTTYPE DHT11 // DHT11 або DHT22 DHT dht(DHTPIN, DHTTYPE); void setup() { Serial.begin(9600); dht.begin(); } void loop() { float h = dht.readHumidity(); float t = dht.readTemperature(); if (isnan(t) || isnan(h)) { Serial.println("Error reading from DHT"); } else { Serial.print("Humidity: "); Serial.print(h); Serial.print(" %\tTemperature: "); Serial.print(t); Serial.println(" *C"); } }
DHT11 оновлює дані кожні 2 секунди. Не опитуйте частіше — отримаєте NaN. Для точніших вимірів використовуйте DHT22 (±0.5°C vs ±2°C у DHT11).

Геркон · INPUT vs INPUT_PULLUP · tone()

Геркон реагує на наближення магніту — замикає контакти. Arduino зчитує стан і активує LED (D13) та Buzzer (D8) через tone(). Вивчаємо два варіанти підключення з порівнянням.

// Варіант 2 (рекомендований): INPUT_PULLUP · Геркон → GND

    Зберіть схему Варіант 2: один контакт геркона → D2, інший → GND. LED із 220 Ом → D13 → GND. Buzzer: + → D8, − → GND. Зовнішній резистор для геркона НЕ потрібен!
  • 2
    Введіть Лістинг «Варіант 2». Зверніть увагу: INPUT_PULLUP замість INPUT і умова reedState == LOW (логіка інвертована).
  • 3
    Завантажте. Піднесіть магніт до геркона — LED засвітиться, buzzer зазвучить на 1000 Гц. Відведіть — система заспокоїться.
  • 4
    Щоб зробити «охорону дверей»: прикріпіть магніт до дверей, геркон до рами. Тривога спрацює при відкритті (магніт відходить). Змініть умову на reedState == HIGH.
Варіант 1 · Геркон + INPUT + 10кОм
const int reedPin = 2; const int ledPin = 13; const int buzzerPin = 8; void setup() { pinMode(reedPin, INPUT); // зовнішній R 10кОм до GND pinMode(ledPin, OUTPUT); pinMode(buzzerPin, OUTPUT); Serial.begin(9600); } void loop() { int reedState = digitalRead(reedPin); if (reedState == HIGH) { // магніт є digitalWrite(ledPin, HIGH); tone(buzzerPin, 1000); Serial.println("Тривога! Магніт виявлено!"); } else { digitalWrite(ledPin, LOW); noTone(buzzerPin); } delay(100); }
Варіант 2 · INPUT_PULLUP (рекомендований!)
const int reedPin = 2; const int ledPin = 13; const int buzzerPin = 8; void setup() { pinMode(reedPin, INPUT_PULLUP); // вбудований ~20кОм pinMode(ledPin, OUTPUT); pinMode(buzzerPin, OUTPUT); Serial.begin(9600); } void loop() { int reedState = digitalRead(reedPin); if (reedState == LOW) { // LOW = замкнено на GND digitalWrite(ledPin, HIGH); tone(buzzerPin, 1000); Serial.println("Контакт замкнено!"); } else { digitalWrite(ledPin, LOW); noTone(buzzerPin); } delay(50); }
Змініть код так, щоб під час тривоги LED блимав (а не горів постійно). Яку команду для затримки потрібно додати всередині блоку if?
