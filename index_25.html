01 · Теоретична частина

Будова та принцип роботи сервоприводу

Що таке сервопривід

Сервопривід — це привід зі зворотним зв'язком для точного керування положенням осі. Усередині: колекторний двигун, редуктор, потенціометр зворотного зв'язку та контролер із PWM-інтерфейсом.

Характеристики SG90 (9g)

Живлення: 3–7.2 В. Крутний момент: 1.2 кг/см при 4.8 В, 1.6 кг/см при 6 В. Кут: 160–180°. Швидкість: 180° за 0.3 с. Струм: 400 мА (без навантаження), до 1 А стартовий.

Дроти підключення

🟤 Коричневий → GND
🔴 Червоний → VCC (5 В)
🟡 Жовтий → Цифровий ШІМ-пін (~)

Два типи серво

Позиційні: керують кутом (0–180°). Найпоширеніші.
Постійного обертання (360°, continuous): керують швидкістю обертання як звичайний мотор з цифровим управлінням.

⚠️ УВАГА! Живлення. Сервопривід споживає до 1А стартового струму. Рекомендується живити від зовнішнього джерела. При 2+ приводах від Arduino може вигоріти захисний діод на платі! Спільна земля між Arduino та джерелом — обов'язкова.
📡 PDM vs PWM — у чому різниця? ▼

Сервопривід керується довжиною імпульсу (PDM — Pulse Duration Modulation), а не скважністю (PWM):

  • 0° → імпульс ~500 мкс
  • 90° → імпульс ~1500 мкс
  • 180° → імпульс ~2500 мкс

Імпульси подаються з частотою ~50 Гц (аналогове серво). При частоті менше 20 Гц — «джиттер», більше 100 Гц — неадекватна поведінка. Цифрові серво допускають до ~300 Гц.

Технічно PDM та PWM — схожі, але серво очікує саме тривалість імпульсу, а не його відсоткову частку від періоду.

🛠 Бібліотека Servo.h — методи та обмеження ▼

Підключення: #include <Servo.h> та Servo myservo;

  • attach(pin) — підключити серво до піна (діапазон 544–2400 мкс)
  • attach(pin, min, max) — з налаштуванням діапазону мкс
  • write(angle) — повернути на кут у градусах (0–180)
  • writeMicroseconds(us) — повернути за довжиною імпульсу
  • detach() — відключити
⚠️ На Arduino Nano бібліотека використовує Timer1 → ШІМ на пінах 9 та 10 недоступний при використанні Servo.h!

Arduino Uno підтримує до 12 сервоприводів, Arduino Mega — до 48.

🔧 Типові проблеми та їх усунення ▼
  • fatal error: Servo.h — бібліотека не встановлена. Завантажте та встановіть через Manage Libraries.
  • Серво не крутиться — перевірте спільну землю між Arduino та серво. Перевірте достатність струму джерела живлення.
  • Серво тріщить і не обертається — неправильно виставлені крайні положення. Замість 0° → 10°, замість 180° → 170°.
  • Перешкоди / тремтіння — підключіть конденсатор 100 мкФ між 5V та GND плати Arduino.
⚙️ Керування серво без бібліотеки — через digitalWrite ▼

Базова програма керування серво через прямі затримки:

#define SERVO_PIN 3 #define POT_PIN 0 void setup() { pinMode(SERVO_PIN, OUTPUT); } void loop() { int pulse = map(analogRead(POT_PIN), 0, 1023, 500, 2500); digitalWrite(SERVO_PIN, HIGH); delayMicroseconds(pulse); // тривалість імпульсу digitalWrite(SERVO_PIN, LOW); delay(20); // 50 Гц }

Обмеження: такий підхід блокує весь loop() і підходить лише для одного серво. Для кількох — використовуйте Servo.h або асинхронний таймер на millis().

02 · Широтно-імпульсна модуляція

ШІМ (PWM) — принцип і застосування

Що таке ШІМ

PWM (Pulse Width Modulation) — цифровий сигнал, що перемикається між HIGH і LOW з фіксованою частотою. Duty Cycle (шпаруватість) — відсоток часу HIGH за один період.

Піни ШІМ Arduino Uno

Позначені ~: піни 3, 5, 6, 9, 10, 11. Частота: ~490 Гц (піни 3,9,10,11) та ~980 Гц (піни 5,6). Функція: analogWrite(pin, 0–255).

Duty Cycle → напруга

0% → 0 В · 25% → 1.25 В · 50% → 2.5 В · 75% → 3.75 В · 100% → 5 В. Значення analogWrite: 0→0%, 128→50%, 255→100%.

Застосування ШІМ

Яскравість LED · швидкість DC-двигуна · сервоприводи (PDM) · звук через tone() · контраст LCD · нагрів елементів · керування вентиляторами.

Приклади використання ШІМ

💡 1. «Дихання» світлодіода — плавна зміна яскравості ▼
int ledPin = 9; void setup() { pinMode(ledPin, OUTPUT); } void loop() { for (int v = 0; v <= 255; v++) { analogWrite(ledPin, v); // збільшення яскравості delay(10); } for (int v = 255; v >= 0; v--) { analogWrite(ledPin, v); // зменшення яскравості delay(10); } }

Duty Cycle плавно змінюється від 0% до 100% і назад → ефект «дихання» LED. Повний цикл займає ~5 секунд.

⚙️ 2. Керування швидкістю DC-двигуна ▼
int motorPin = 6; void setup() { pinMode(motorPin, OUTPUT); } void loop() { analogWrite(motorPin, 128); // ~50% потужності delay(2000); analogWrite(motorPin, 255); // 100% потужності delay(2000); }

ШІМ-сигнал на вхід драйвера двигуна (L293D, L298N) регулює середню напругу → швидкість обертання. Ніколи не підключайте двигун напряму до Arduino!

Завдання 2 · Ручне керування потенціометром (Project 14-2)

📌 Мета: Обертання ручки потенціометра визначає кут повороту вала сервоприводу в реальному часі. Результати виводяться в Serial Monitor.

1

Підключіть потенціометр до A0

Крайні виводи потенціометра → GND та 5V. Середній вивід (мітка/wiper) → A0. Серво підключіть до піна ~11 як у Завданні 1.

2

Завантажте Лістинг 14-2 та відкрийте Serial Monitor

Спостерігайте значення analogRead(A0) (0–1023) та поточний кут серво (0–255, потім поділений на 4 → 0–63). Зверніть увагу: analogRead/4 дає 0–255, а не 0–180!

3

Порівняйте з функцією map()

Змініть рядок servo.write(analogRead(A0)/4) на servo.write(map(analogRead(A0),0,1023,0,180)) — тепер кут буде в правильному діапазоні 0–180°.

Лістинг 14-2 · Потенціометр керує серво

#include <Servo.h> Servo servo; void setup() { servo.attach(11); pinMode(A0, INPUT); Serial.begin(9600); } void loop() { servo.write(analogRead(A0) / 4); // значення 0–255 → серво Serial.println(analogRead(A0)); // значення потенціометра Serial.println(analogRead(A0) / 4); // значення на вхід серво Serial.println(); // порожній рядок delay(1000); } // Покращена версія з map() — правильний діапазон 0..180°: // servo.write(map(analogRead(A0), 0, 1023, 0, 180));
✅ Порада: використовуйте map(analogRead(A0), 0, 1023, 0, 180) для точної відповідності кута потенціометру — повний оберт = повний кут серво.

Завдання 4 · RFID-замок на сервоприводі (Project 14-4)

📌 Мета: прототип системи відкриття дверей RFID-карткою. Серво відкриває «двері» при збігу UID, зелений LED + звуковий сигнал підтверджують доступ.

Нам знадобиться:

🟦Arduino UNO
💻ПК + Arduino IDE
🔲Макетна плата
⚙1 сервопривід
🔌З'єднувальні дроти
🔴🟢2 LED (червоний/зелений)

1

Підключіть RFID MFRC522 через SPI

SS → D10, RST → D9, MOSI → D11, MISO → D12, SCK → D13, VCC → 3.3V (!), GND → GND. Зверніть увагу: MFRC522 живиться від 3.3 В, не 5 В!

2

Встановіть бібліотеку MFRC522

Arduino IDE → Manage Libraries → знайдіть «MFRC522» (Miguel Balboa) → Install. Також потрібна стандартна бібліотека SPI.h.

3

Визначте UID вашої картки

Завантажте приклад File → Examples → MFRC522 → DumpInfo. Прикладіть картку до зчитувача, скопіюйте UID із Serial Monitor і замініть в лістингу значення 49481524 на свій UID.

4

Доповніть лістинг — замініть ***

Рядки *** у лістингу — місця для вашого коду. Підказки: tone(buzzerPin, 1000, 200) для звуку, servo.write(90) для відкриття, delay(3000) пауза, servo.write(0) для закриття.

Лістинг 14-4 · RFID-замок (з підказками)

#include <Servo.h> #include <SPI.h> #include <MFRC522.h> // бібліотека RFID #define SS_PIN 10 // SS → D10 #define RST_PIN 9 // RST → D9 #define LED_G 5 // зелений LED #define LED_R 6 // червоний LED #define BUZZER 4 // зумер MFRC522 mfrc522(SS_PIN, RST_PIN); unsigned long uidDec, uidDecTemp; Servo servo; void setup() { pinMode(LED_G, OUTPUT); pinMode(LED_R, OUTPUT); digitalWrite(LED_R, HIGH); // спочатку — червоний Serial.begin(9600); Serial.println("Waiting for card..."); SPI.begin(); mfrc522.PCD_Init(); servo.attach(3); servo.write(0); // початковий стан — зачинено } void loop() { if (!mfrc522.PICC_IsNewCardPresent()) return; if (!mfrc522.PICC_ReadCardSerial()) return; uidDec = 0; for (byte i = 0; i < mfrc522.uid.size; i++) { uidDecTemp = mfrc522.uid.uidByte[i]; uidDec = uidDec * 256 + uidDecTemp; } Serial.print("Card UID: "); Serial.println(uidDec); // виводимо UID картки if (uidDec == 49481524) { // ← замініть на свій UID! digitalWrite(LED_R, LOW); tone(BUZZER, 1000, 200); // звуковий сигнал — доступ відкрито digitalWrite(LED_G, HIGH); servo.write(90); // відкрити двері delay(3000); // тримати 3 секунди servo.write(0); // закрити двері tone(BUZZER, 500, 200); // сигнал — закрито mfrc522.PICC_HaltA(); delay(5000); // пауза між зчитуваннями } digitalWrite(LED_G, LOW); digitalWrite(LED_R, HIGH); }

Лістинг 14-5 · Шість серво — асинхронне керування

#include <Servo.h> Servo s[6]; // масив із 6 об'єктів Servo int pins[] = {3, 5, 6, 9, 10, 11}; // ШІМ-піни int angles[6] = {0,0,0,0,0,0}; int dirs[6] = {1,1,1,1,1,1}; uint32_t tmr; void setup() { for (int i = 0; i < 6; i++) s[i].attach(pins[i]); } void loop() { // асинхронний таймер — не блокує loop() if (millis() - tmr >= 20) { tmr = millis(); for (int i = 0; i < 6; i++) { angles[i] += dirs[i]; // крок if (angles[i] >= 180 || angles[i] <= 0) dirs[i] = -dirs[i]; // реверс s[i].write(angles[i]); } } // тут можна виконувати інший код без блокування! }
✅ Асинхронний підхід через millis() дозволяє одночасно керувати 6 серво і виконувати інший код — без зупинки програми на delay().
05 · Контрольні питання

Тест — 10 питань

1. Яка тривалість PDM-імпульсу відповідає положенню 90° сервоприводу SG90?
A~500 мкс
B~1500 мкс
C~2500 мкс
D~3000 мкс
Діапазон: ~500 мкс (0°) → ~1500 мкс (90°) → ~2500 мкс (180°). Бібліотека Servo.h за замовчуванням використовує 544–2400 мкс.
2. Яка рекомендована частота PDM-сигналу для аналогового сервоприводу?
A10 Гц
B~50 Гц
C490 Гц
D1000 Гц
~50 Гц (период 20 мс). При менше 20 Гц — «джиттер». При більше 100 Гц — неадекватна поведінка аналогового серво. Цифрові серво допускають до ~300 Гц.
3. Що означає Duty Cycle = 25% у ШІМ при напрузі 5 В?
AСередня напруга 0.5 В
BСередня напруга 1.25 В
CСередня напруга 2.5 В
DНапруга завжди 5 В
Duty Cycle = 25% → сигнал HIGH 25% часу. Середня напруга = 5В × 0.25 = 1.25 В. Формула: V_avg = V_max × (DC% / 100).
4. Яке значення analogWrite() відповідає Duty Cycle ≈ 50%?
A50
B100
C128
D512
analogWrite(pin, value) приймає 0–255. 128 ÷ 255 ≈ 50.2%. Формула: value = DC% × 255 / 100. Значення 255 → 100%, 0 → 0%.
5. Які піни Arduino Uno НЕ підтримують ШІМ?
AПіни 3, 5, 6, 9, 10, 11
BПіни 0, 1, 2, 4, 7, 8, 12, 13
CУсі цифрові піни підтримують ШІМ
DЛише аналогові піни A0–A5
ШІМ-піни на Uno позначені «~»: 3, 5, 6, 9, 10, 11. Решта цифрових пінів (0,1,2,4,7,8,12,13) та аналогові не підтримують analogWrite().
6. Яка функція бібліотеки Servo.h повертає вал на заданий кут у градусах?
Aservo.rotate(angle)
Bservo.setAngle(angle)
Cservo.write(angle)
Dservo.move(angle)
servo.write(angle) — основна функція: приймає кут 0–180° і генерує відповідний PDM-імпульс. servo.writeMicroseconds(us) — точніше керування довжиною імпульсу.
7. Чому сервопривід рекомендується живити від зовнішнього джерела?
AБібліотека Servo.h вимагає 12 В
BСерво споживає до 1 А стартового струму, що може пошкодити Arduino
CСерво не може працювати від 5 В
DArduino не підтримує ШІМ для серво
Стартовий струм SG90 до 1 А. Arduino може видавати лише ~500 мА. При 2+ приводах може вигоріти захисний діод. При зовнішньому живленні — спільна земля обов'язкова!
8. Яке обмеження бібліотеки Servo.h на Arduino Nano?
AМаксимум 1 серво
BНе підтримує write() більше 90°
CЗаймає Timer1 → ШІМ на пінах 9 та 10 недоступний
DВимагає окремого живлення 12 В
На Arduino Nano (і Uno) бібліотека Servo.h використовує Timer1, що відключає апаратний ШІМ на пінах 9 та 10. Функція analogWrite(9,...) або analogWrite(10,...) не працюватиме.
9. Як правильно перетворити значення A0 (0–1023) у кут серво (0–180°)?
AanalogRead(A0) / 10
BanalogRead(A0) / 4
Cmap(analogRead(A0), 0, 1023, 0, 180)
DanalogRead(A0) * 180 / 5
map(analogRead(A0), 0, 1023, 0, 180) — правильне масштабування. analogRead/4 дає 0–255, що виходить за межі 0–180°. Функція map() виконує лінійне перетворення діапазонів.
10. Яка перевага асинхронного керування серво через millis() перед delay()?
AТочніший кут повороту
BМенше споживання струму
CНе блокує loop() — можна одночасно керувати кількома пристроями
DПідтримує більший кут повороту
millis()-таймер не зупиняє виконання програми, на відміну від delay(). Це дозволяє одночасно: керувати 6 серво, читати датчики, перевіряти кнопки тощо. Ключова техніка для складних проєктів.
