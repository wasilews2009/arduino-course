Теоретичний матеріал

Цикл for — відома кількість повторень

Цикл for виконується задану кількість разів. Підходить, коли заздалегідь відомо, скільки ітерацій потрібно — наприклад, 5 разів блимнути LED або перебрати масив.

for (ініціалізація; умова; зміна) {
// команди, що повторюються
}
ініціалізація // byte i = 0; — створюємо змінну-лічильник
умова // i <= 5; — цикл триває, поки умова true
зміна // i++; — збільшуємо лічильник
💡 Приклад: 5 разів блимнути LED на піні 13+
for — 5 блимань
void setup() { pinMode(13, OUTPUT); } void loop() { for (byte i = 0; i <= 5; i++) { // і від 0 до 5 включно digitalWrite(13, HIGH); delay(500); digitalWrite(13, LOW); delay(500); } // після 6 ітерацій (i=0,1,2,3,4,5) — вихід, loop() повторюється }
Крок зміни може бути будь-яким: i+=2 — крок 2; i-- — зворотній відлік; i = i*2 — подвоєння.
🔧 Використання for у setup() для масового налаштування+
for в setup() — налаштування 5 пінів
void setup() { // Без for: 5 рядків. З for — 1 рядок! for (int pin = 8; pin <= 12; pin++) { pinMode(pin, OUTPUT); // піни 8,9,10,11,12 } }
Цикл while — умовне повторення

Цикл while виконується безперервно, поки умова в дужках є true. Підходить, коли кількість ітерацій невідома заздалегідь — наприклад, поки давач не дасть потрібне значення.

byte i = 0; // ← змінну оголошуємо ДО циклу
while (i < 5) {
// команди
i++; // ← обов'язково змінювати змінну!
}
Якщо забути i++ всередині while — цикл ніколи не завершиться. Arduino «зависне» у нескінченному циклі!
💡 while з датчиком — чекаємо доки не порогове значення+
while — чекаємо натискання кнопки
void loop() { // Чекаємо, поки не натиснуть кнопку на D2 while (digitalRead(2) == LOW) { // нічого не робимо — просто чекаємо } // Після виходу з while — кнопку натиснуто digitalWrite(13, HIGH); // вмикаємо LED }
Цикл do...while — хоча б один раз

Відмінність від while: тіло циклу виконується мінімум один раз, і лише потім перевіряється умова. Корисно, коли дія повинна відбутися хоча б один раз незалежно від умови.

do...while — зчитуємо датчик хоча б раз
int water; // змінну оголошуємо ДО циклу do { digitalWrite(13, HIGH); // LED вмикається ЗАВЖДИ мінімум раз water = analogRead(A0); // читаємо датчик } while (water < 5); // умова перевіряється ПІСЛЯ digitalWrite(13, LOW);
Порівняння трьох циклів:
for — знаємо кількість ітерацій заздалегідь
while — умова перевіряється до виконання (може не виконатись жодного разу)
do...while — умова перевіряється після (мінімум 1 виконання гарантоване)
break та goto — керований вихід з циклу

break — миттєво виходить з поточного циклу. Виконання продовжується з першого рядка після фігурних дужок циклу.

goto — переходить до виконання коду з будь-якого міченого місця в програмі.

⚡ Приклади break та goto+
break — зупинити цикл при умові
for (int i = 0; i < 100; i++) { if (digitalRead(2) == HIGH) { break; // кнопку натиснуто — виходимо негайно } digitalWrite(13, !digitalRead(13)); delay(100); } // програма продовжується тут
goto — перехід за міткою
startLoop: // мітка digitalWrite(13, HIGH); delay(500); digitalWrite(13, LOW); delay(500); if (digitalRead(2) == LOW) goto startLoop; // повертаємось до мітки
goto ускладнює читання коду. Використовуйте лише коли break недостатній (вихід з вкладених циклів).
RGB-світлодіод — три в одному

RGB-світлодіод — це три незалежних LED (червоний, зелений, синій) в одному корпусі. Змішуючи їх яскравість через ШІМ (analogWrite, 0–255), отримуємо будь-який колір.

ТипСпільний контактПідключення спільногоЛогіка яскравості
Спільний катод (CC)GND (−)→ GNDanalogWrite 0=вимк, 255=макс
Спільний анод (CA)VCC (+)→ 5VanalogWrite 0=макс, 255=вимк ⚠
Завжди встановлюйте резистор 220–330 Ом на кожен кольоровий канал! Без резистора — LED згорить.

Змішування кольорів (R, G, B = 0–255)

Синій00255

Голубий (Cyan)0255255

Колірна модель HSB (HSV): Hue (відтінок 0–360°), Saturation (насиченість 0–1), Brightness (яскравість 0–1). Функція HSBtoRGB() перетворює HSB у RGB. Це дозволяє плавно переходити крізь увесь спектр веселки, просто змінюючи Hue від 0 до 360.

Project 9-1 · analogWrite · random()
Знайомство з RGB — випадкові кольори

Базовий проєкт з RGB: кожну секунду світлодіод змінює колір на випадковий, генеруючи random(0,255) для кожного з трьох каналів. Використовуються ШІМ-піни (~) і функція analogWrite().

Необхідне обладнання
Arduino UNO
RGB LED (CC або CA)
3 резистори 220–330 Ом
Макетна плата
5 дротів «тато-тато»
// Принципова схема · RGB LED (спільний катод) · Піни 9,10,11
У Лістингу 9-1 використовується #define LEDG 13 — але пін 13 не має ШІМ (~)! Для плавного регулювання яскравості зеленого використовуйте пін 10 або 11. Пін 13 лише дає HIGH/LOW.
  • 1
    Зберіть схему. Знайдіть найдовшу ніжку (спільний катод) → GND. Три коротших → через резистори 220 Ом → D9(R), D10(G), D11(B).
  • 2
    Введіть Лістинг 9-1. Зверніть увагу на #define для пінів та random(0,255) для кожного каналу.
  • 3
    Завантажте. Кожну секунду LED змінює колір на новий випадковий. Зафіксуйте у звіті 5 різних кольорових комбінацій (R, G, B).
Лістинг 9-1 · Випадкові кольори RGB
#define LEDR 11 // ← рекомендується 9 (ШІМ) #define LEDG 13 // ← рекомендується 10 (ШІМ)! #define LEDB 12 // ← рекомендується 11 (ШІМ) void setup() { pinMode(LEDR, OUTPUT); pinMode(LEDG, OUTPUT); pinMode(LEDB, OUTPUT); } int r = 0, g = 0, b = 0; void loop() { r = random(0, 255); // 0–254 g = random(0, 255); b = random(0, 255); analogWrite(LEDR, r); analogWrite(LEDG, g); analogWrite(LEDB, b); delay(1000); }

Project 9-3 · for · setColor() · Плавний перехід
Світлофор настрою — веселка через цикли for

Плавне перетікання кольорів: Червоний → Зелений → Синій → Червоний. Три цикли for, кожен з 256 кроками, поступово зменшують один канал і збільшують інший. Допоміжна функція setColor(r,g,b).

Необхідне обладнання
Arduino UNO
RGB LED
3 резистори 220–330 Ом
Макетна плата + дроти
Принцип плавного переходу: Щоб перейти від Червоного до Зеленого: крок за кроком зменшуємо R (255→0) і збільшуємо G (0→255). setColor(255-i, i, 0) — при i=0: Чистий Червоний. При i=127: Жовтий. При i=255: Чистий Зелений.
  • 1
    Схема незмінна від Завдання 1. Піни: redPin=9, greenPin=10, bluePin=11.
  • 2
    Введіть Лістинг 9-23. Зверніть увагу на функцію setColor() — вона отримує 3 значення і одразу записує їх через analogWrite().
  • 3
    delay(10) × 256 кроків = 2.56 секунди на перехід. Змініть затримку і перевірте швидкість анімації.
Лістинг 9-23 · Плавний перехід кольорів
const int redPin = 9, greenPin = 10, bluePin = 11; void setup() { pinMode(redPin, OUTPUT); pinMode(greenPin, OUTPUT); pinMode(bluePin, OUTPUT); } void loop() { // Червоний → Зелений for (int i = 0; i <= 255; i++) { setColor(255 - i, i, 0); delay(10); } // Зелений → Синій for (int i = 0; i <= 255; i++) { setColor(0, 255 - i, i); delay(10); } // Синій → Червоний for (int i = 0; i <= 255; i++) { setColor(i, 0, 255 - i); delay(10); } } void setColor(int r, int g, int b) { analogWrite(redPin, r); analogWrite(greenPin, g); analogWrite(bluePin, b); }

Project 9-4 · 3× потенціометр · analogRead → map → analogWrite
Аналоговий мікшер кольорів

Три потенціометри керують трьома каналами RGB. Повертаєте ручку — колір змінюється в реальному часі. analogRead(A0) (0–1023) → map() → (0–255) → analogWrite().

Необхідне обладнання
Arduino UNO
RGB LED
3 резистори 220 Ом
3 потенціометри 10 кОм
Макетна плата + дроти
// Принципова схема · 3 потенціометри + RGB LED
  • 1
    Підключіть потенціометри: ліва ніжка→GND, права→5V, середня (вихід)→A0, A1, A2 відповідно.
  • 2
    Підключіть RGB LED через резистори 220 Ом до пінів D9, D10, D11 (ШІМ-піни ~). GND→GND.
  • 3
    Введіть Лістинг 9-24. Функція map(valR, 0, 1023, 0, 255) масштабує АЦП 10-біт до ШІМ 8-біт.
  • 4
    Для спільного аноду (CA): змініть map(val, 0, 1023, 0, 255) на map(val, 0, 1023, 255, 0) — інверсія логіки.
Лістинг 9-24 · Мікшер кольорів (потенціометри)
const int potRed=A0, potGreen=A1, potBlue=A2; const int ledRed=9, ledGreen=10, ledBlue=11; void setup() { pinMode(ledRed, OUTPUT); pinMode(ledGreen, OUTPUT); pinMode(ledBlue, OUTPUT); } void loop() { int valR = analogRead(potRed); // 0–1023 int valG = analogRead(potGreen); int valB = analogRead(potBlue); int bR = map(valR, 0, 1023, 0, 255); // → 0–255 int bG = map(valG, 0, 1023, 0, 255); int bB = map(valB, 0, 1023, 0, 255); analogWrite(ledRed, bR); analogWrite(ledGreen, bG); analogWrite(ledBlue, bB); delay(10); }
Для спільного аноду: map(val, 0, 1023, 255, 0) — потенціометр на 0 → максимум яскравості.

Project 9-2 · switch/case · HSBtoRGB() · Веселка
Веселка — HSB колірна модель (завдання на «зірочку»)

Замість простих переходів між 3 кольорами — повний безперервний спектр через HSB колірну модель. Hue (відтінок) плавно змінюється 0°→360° → усі кольори веселки. Функція HSBtoRGB() використовує switch/case для 6 секторів кольорового колеса.

Ключові змінні Лістингу 9-2:
numColors = 255 — кількість кроків (більше = плавніше)
animationDelay = 10 — мс між кроками
saturation = 1.0 — насиченість (0 = сірий, 1 = чистий колір)
brightness = 0.05 — яскравість (0.05 = дуже тьмяно, 1.0 = максимально)
  • 1
    Схема незмінна. Піни: PIN_RED=10, PIN_GREEN=11, PIN_BLUE=9.
  • 2
    Введіть повний Лістинг 9-2 включаючи функцію HSBtoRGB(). Зверніть увагу на бітові операції: color >> 16 & 255 — отримати червону складову з упакованого long.
  • 3
    Збільшіть brightness з 0.05 до 0.3 і переконайтесь у різниці яскравості. Поексперементуйте з saturation: значення 0.5 дасть пастельні відтінки.
Додайте кнопку на D2, яка перемикає режими:
1) Повільна веселка (animationDelay=30, numColors=255)
2) Швидка веселка (animationDelay=2, numColors=100)
3) Фіксований колір (пауза на поточному Hue).
