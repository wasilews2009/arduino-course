Project 5-1 · Аналоговий вхід · if-оператор
«Розумний світильник» — фоторезистор

Фоторезистор підключається до аналогового входу A0 у схемі подільника напруги. Arduino зчитує значення 0–1023 і через умовний оператор if вмикає або вимикає LED залежно від освітленості.

Необхідне обладнання
Arduino UNO
Фоторезистор (LDR)
2 резистори 220 Ом
Світлодіод
Макетна плата
5 дротів «тато-тато»
📖 Принцип роботи фоторезистора+

Фоторезистор — напівпровідниковий прилад, у якого під впливом світла знижується електричний опір (явище фотопровідності). При освітленні фотони вибивають електрони з валентної зони до зони провідності, збільшуючи кількість рухомих носіїв заряду.

  • У темряві: опір великий (10–100 кОм) → напруга на A0 мала
  • На світлі: опір малий (100–1000 Ом) → напруга на A0 велика
  • analogRead(A0) повертає значення від 0 (0В) до 1023 (5В)
// Принципова схема · подільник напруги LDR + LED
  • 1
    Зберіть схему подільника: від 5В → LDR → вузол → 220 Ом → GND. Точка між LDR і резистором — на A0. Другий резистор 220 Ом послідовно зі світлодіодом від D9 до GND.
  • 2
    Введіть Лістинг 5-1. Зверніть увагу: #define SENSOR A0 та #define LED 9 — директиви препроцесора, не займають SRAM.
  • 3
    Завантажте. Затуліть фоторезистор рукою — LED має засвітитися (value<500). Відкрийте — LED гасне.
  • 4
    Відкрийте Serial Monitor (9600 бод). Виведіть значення value командою Serial.println(value) для розуміння порогу.
Лістинг 5-1 · Фоторезистор → LED автоматика
#define SENSOR A0 // аналоговий вхід з фоторезистором #define LED 9 // цифровий вихід зі світлодіодом unsigned int value = 0; // 0..1023, тільки додатні void setup() { pinMode(LED, OUTPUT); pinMode(SENSOR, INPUT); Serial.begin(9600); } void loop() { value = analogRead(SENSOR); // зчитуємо АЦП: 0–1023 Serial.println(value); // виводимо для налаштування порогу if (value < 500) digitalWrite(LED, HIGH); // темно → LED ON if (value > 500) digitalWrite(LED, LOW); // світло → LED OFF }
Поріг 500 — середина діапазону 0–1023. Відкрийте Serial Monitor і перевірте реальні значення у вашому приміщенні, щоб налаштувати точніший поріг.

Project 5-3 · Аналоговий давач · Serial Monitor
Вимірювання температури давачем TMP36

Аналоговий давач температури TMP36 (або LM35) перетворює температуру в напругу: 10 мВ/°C зі зміщенням 500 мВ. Arduino зчитує АЦП і конвертує в °C та °F. Дані виводяться в Serial Monitor.

Необхідне обладнання
Arduino UNO
Давач TMP36 (або LM35)
Макетна плата
5 дротів «тато-тато»
📐 Формули розрахунку температури TMP36+
Ланцюжок перетворень:
1. voltage = reading × 5.0 / 1024.0 // АЦП → Вольти
2. tempC = (voltage − 0.5) × 100 // В → °C (offset 500мВ)
3. tempF = tempC × 9.0 / 5.0 + 32 // °C → °F
TMP36 при 25°C видає 750 мВ. Чутливість: +10 мВ на кожен °C. Діапазон: −40°C … +125°C.
// Принципова схема · TMP36 → A0 → Serial Monitor
  • 1
    Підключіть TMP36: лівий пін (плоска сторона до вас) → 5V, середній → A0, правий → GND. Не переплутайте! Перегрів від зворотного підключення може пошкодити давач.
  • 2
    Введіть Лістинг 5-3. Зверніть увагу: reading × 5.0 / 1024.0 — конвертація АЦП у вольти (5.0 — референсна напруга Arduino).
  • 3
    Завантажте. Відкрийте Serial Monitor (9600 бод). Ви маєте побачити напругу (~0.75В при 25°C), температуру °C та °F.
  • 4
    Перевірте давач: потримайте в руках (температура зросте на 2–5°C) або піднесіть до нагрітого предмета.
Лістинг 5-3 · Термометр TMP36 → Serial Monitor
int sensorPin = 0; // A0 — аналоговий пін TMP36 void setup() { Serial.begin(9600); } void loop() { int reading = analogRead(sensorPin); // 0–1023 // 1. АЦП → Вольти float voltage = reading * 5.0 / 1024.0; // 2. Напруга → °C (10мВ/°C, зміщення 500мВ) float tempC = (voltage - 0.5) * 100; // 3. °C → °F float tempF = tempC * 9.0 / 5.0 + 32.0; Serial.print(voltage); Serial.println(" Вольти"); Serial.print(tempC); Serial.println(" градуси C"); Serial.print(tempF); Serial.println(" градуси F"); delay(1000); }
При підключенні TMP36 стежте за орієнтацією! Плоска сторона корпусу — до читача: лівий пін VCC, середній OUT, правий GND.

Project 5-4 · Аналоговий давач · tone()
Сигналізація вологості ґрунту

Датчик вологості ґрунту YL-69 / FC-28 вимірює електричний опір між щупами: сухий ґрунт → великий опір → великий АЦП. Якщо значення перевищує поріг — п'єзодинамік подає звукову тривогу.

Необхідне обладнання
Arduino UNO
Датчик вологості YL-69 / FC-28
П'єзодинамік (buzzer)
Макетна плата
З'єднувальні дроти
Важливо: схема у завданні містить помилку! Перевірте підключення перед збиранням: аналоговий вихід датчика (A) → A0, VCC → 5V, GND → GND. Buzzer сигнальний пін → D8.
// Принципова схема · Датчик ґрунту + Buzzer (ПРАВИЛЬНА)
  • 1
    Підключіть: аналоговий вихід A датчика → A0 Arduino. VCC → 5V. GND → GND. Buzzer: + → D8, − → GND.
  • 2
    Введіть Лістинг 5-4. Поріг threshold = 750 — початкове значення для калібрування.
  • 3
    Калібрування: відкрийте Serial Monitor (9600 бод). Перевірте значення: щуп у повітрі (~1023), у воді (~200–400). Виберіть поріг між ними.
  • 4
    Перевірте: вийміть щуп з вологого ґрунту — buzzer має подати сигнал 1000 Гц.
Лістинг 5-4 · Сигналізація вологості ґрунту
const int sensorPin = A0; // аналоговий вихід датчика const int buzzerPin = 8; // п'єзодинамік int threshold = 750; // поріг (калібрується!) void setup() { pinMode(buzzerPin, OUTPUT); Serial.begin(9600); Serial.println("Система моніторингу вологості запущена."); } void loop() { int moistureValue = analogRead(sensorPin); Serial.print("АЦП: "); Serial.println(moistureValue); if (moistureValue > threshold) { // ґрунт сухий tone(buzzerPin, 1000); // звуковий сигнал 1 кГц Serial.println("УВАГА! Потрібен полив!"); } else { noTone(buzzerPin); // ґрунт вологий — тиша } delay(1000); }
Значення АЦП: 1023 = абсолютно сухо, ~300–400 = повністю мокро. Встановіть поріг між «допустимо вологий» та «критично сухий» для вашого типу рослини.

Project 5-5 · map() · tone() · Serial Monitor
Електронний теремін — музика від освітлення

Фоторезистор у схемі подільника напруги реагує на тінь від руки. Функція map() перетворює значення освітленості (0–1023) у звукову частоту (120–1500 Гц). П'єзодинамік відтворює ноти — теремін готовий!

Необхідне обладнання
Arduino UNO
Фоторезистор (LDR)
Резистор 10 кОм
П'єзодинамік (buzzer)
Макетна плата
З'єднувальні дроти
🎵 Функція map() — масштабування діапазонів+
Синтаксис map()
map(value, fromLow, fromHigh, toLow, toHigh)
Пропорційно переводить value з діапазону [fromLow..fromHigh]
до діапазону [toLow..toHigh]
// Приклад тереміна:
pitch = map(lightValue, 100, 900, 120, 1500);
// 100→120 Гц (темно, рука близько); 900→1500 Гц (світло)
// Принципова схема · Теремін: LDR + 10кОм + Buzzer
  • 1
    Зберіть подільник: 5V → LDR → вузол A0 → 10 кОм → GND. Buzzer: + → D8, − → GND.
  • 2
    Введіть Лістинг 5-5. Ключова функція: pitch = map(lightValue, 100, 900, 120, 1500) — перетворює освітленість у частоту.
  • 3
    Завантажте. Проведіть рукою над фоторезистором — тональність змінюється! Serial Monitor показує значення LDR та розраховану частоту.
  • 4
    Тестування: змініть межі в map() для іншого музичного діапазону. При дуже малому світлі (<50) — buzzer вимикається.
Лістинг 5-5 · Електронний теремін
const int sensorPin = A0; // фоторезистор const int buzzerPin = 8; // п'єзодинамік int lightValue; // поточний рівень освітлення int pitch; // частота звуку в Гц void setup() { pinMode(buzzerPin, OUTPUT); Serial.begin(9600); } void loop() { lightValue = analogRead(sensorPin); // зазвичай 200–900 // Масштабуємо освітленість у частоту 120–1500 Гц pitch = map(lightValue, 100, 900, 120, 1500); if (lightValue < 50) { noTone(buzzerPin); // повна темрява → тиша } else { tone(buzzerPin, pitch); // генеруємо звук } Serial.print("Світло: "); Serial.print(lightValue); Serial.print(" -> Частота: "); Serial.println(pitch); delay(20); // швидке оновлення для плавного звуку }
Змініть межі map(lightValue, 100, 900, 120, 1500) та реалізуйте проміжні значення, що відповідають нотам музичної гами.
