Project 24-1 · OLED + BMP180
Домашня метеостанція

Метеостанція на базі датчика тиску/температури BMP180 та графічного OLED-дисплея 0.96" (128×64). Обидва пристрої з'єднуються через єдину шину I²C з різними адресами.

Необхідне обладнання
Arduino UNO
OLED 0.96" I2C 128×64
Датчик BMP180
Макетна плата
З'єднувальні дроти
USB-кабель + ПК
Ключовий принцип: обидва пристрої підключаються паралельно до одних і тих самих пінів I²C. Шина I²C ідентифікує кожен пристрій за унікальною адресою.
OLED SSD1306 адреса: 0x3C    BMP180 адреса: 0x77

  • 1
    Зберіть схему на макетній платі. Підключіть SDA/SCL обох пристроїв паралельно до A4/A5 Arduino.
  • 2
    У Fritzing відтворіть принципову схему проєкту та збережіть файл.
  • 3
    Встановіть бібліотеки: Adafruit_SSD1306 Adafruit_GFX Adafruit_BMP085 через менеджер бібліотек IDE.
  • 4
    Введіть або відкопіюйте Лістинг 24-1 у Arduino IDE. Скомпілюйте та завантажте.
  • 5
    Перевірте що на OLED відображаються: температура (великим шрифтом), тиск у мм рт.ст., висота та іконка сонця.
Лістинг 24-1 · Метеостанція BMP180 + OLED
#include <Wire.h> #include <Adafruit_GFX.h> #include <Adafruit_SSD1306.h> #include <Adafruit_BMP085.h> // BMP085 сумісна з BMP180 #define SCREEN_WIDTH 128 #define SCREEN_HEIGHT 64 Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1); Adafruit_BMP085 bmp; void setup() { Serial.begin(9600); if (!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) { Serial.println(F("SSD1306 allocation failed")); for(;;); // зупиняємо, якщо екран не знайдено } if (!bmp.begin()) { Serial.println("BMP180 not found!"); while(1) {} } display.clearDisplay(); display.setTextColor(WHITE); } void loop() { float temperature = bmp.readTemperature(); int32_t pressurePa = bmp.readPressure(); float pressureMmHg = pressurePa * 0.0075006; // Па → мм рт.ст. float altitude = bmp.readAltitude(); display.clearDisplay(); // ── Заголовок ──────────────────────────────── display.setTextSize(1); display.setCursor(0, 0); display.print("WEATHER STATION v1.0"); display.drawFastHLine(0, 10, 128, WHITE); // ── Температура (великий шрифт) ─────────────── display.setCursor(0, 16); display.setTextSize(2); display.print(temperature, 1); display.setTextSize(1); display.print(" C"); // ── Тиск ───────────────────────────────────── display.setTextSize(1); display.setCursor(0, 36); display.print("Pressure: "); display.setCursor(0, 46); display.print((int)pressureMmHg); display.print(" mmHg"); // ── Висота ─────────────────────────────────── display.setCursor(0, 56); display.print("Alt: "); display.print(altitude, 0); display.print(" m"); // ── Іконка: сонце (коло + промінчики) ──────── display.drawCircle(105, 35, 12, WHITE); display.drawLine(105, 18, 105, 14, WHITE); display.drawLine(105, 52, 105, 56, WHITE); display.drawLine(88, 35, 84, 35, WHITE); display.drawLine(122, 35, 126, 35, WHITE); display.display(); // передаємо буфер на фізичний екран delay(1000); }
Про OLED: команди print(), drawCircle() тощо записують у відеобуфер у SRAM. Лише display.display() фізично передає цей буфер через I²C на екран (128×64 = 1024 байти = 1 КБ SRAM!).
Використовуючи display.drawPixel(), побудуйте на екрані графік зміни атмосферного тиску за останню хвилину. Оновлюйте кожні 5–10 секунд.

Project 7-1 · KY-008 + Keypad
Лазерна сигналізація «Місія нездійсненна»

Прототип лазерної охоронної системи: лазерний промінь від модуля KY-008 потрапляє на фоточутливий давач. При розмиканні — пищалка подає сигнал тривоги. Для знімання сигналізації — введіть пароль на 4×4 мембранній клавіатурі.

Необхідне обладнання
Arduino UNO
Модуль лазера KY-008
Модуль давача лазера
4×4 мембранна клавіатура
Динамік-пищалка
2 макетні плати
Дроти «тато-тато» (16 шт.)
Світлодіод
Алгоритм роботи:
Лазер вмикається → промінь потрапляє на давач (A0 > 500) → нормальний стан.
Промінь розімкнуто (A0 < 500) → detection = true → тривога → чекаємо «*» → вводимо пароль «1366» → вимкнення.

Підключення компонентів

Давач лазераA (Analog)A0analogRead()

Keypad стовпціC1–C3D4, D3, D2OUTPUT
// Принципова схема · Лазер + Давач + Keypad
  • 1
    Розмістіть модуль лазера і давача точно навпроти одне одного. Ретельно вирівняйте їх на двох макетних платах — інакше система починатиме пищати одразу після завантаження.
  • 2
    Підключіть сигнальний пін лазера до D9, давача — до A0, пищалки — до D12. Keypad: рядки до D5–D8, стовпці до D2–D4.
  • 3
    Встановіть бібліотеку Keypad через менеджер бібліотек IDE. Введіть Лістинг 7-1.
  • 4
    Завантажте скетч. Перевірте: при перекритті лазерного проміня пальцем — пищалка сигналізує. Після натискання «*» — вводьте пароль 1366.
  • 5
    Перевірте поведінку при неправильному паролі та при повторному перериванні лазера.
Лістинг 7-1 · Лазерна сигналізація з паролем
#include <Keypad.h> #define Rec 0 // A0 — давач лазера #define Play 12 // D12 — пищалка #define Laser 9 // D9 — лазер bool detection = false; short a = 0; short code[4] = {'1', '3', '6', '6'}; // пароль '1366' const byte numRows = 4, numCols = 3; char keymap[numRows][numCols] = { {'1','2','3'}, {'4','5','6'}, {'7','8','9'}, {'*','0','#'} }; byte rowPins[numRows] = {8,7,6,5}; byte colPins[numCols] = {4,3,2}; Keypad myKeypad = Keypad(makeKeymap(keymap), rowPins, colPins, numRows, numCols); void setup() { pinMode(Laser, OUTPUT); digitalWrite(Laser, HIGH); // вмикаємо лазер відразу pinMode(Play, OUTPUT); delay(2000); // час вирівняти промінь } void loop() { short Detect = analogRead(Rec); if (Detect < 500) detection = true; // промінь розімкнуто if (detection) { tone(Play, 1000); delay(50); tone(Play, 200); delay(50); char keypressed = myKeypad.getKey(); if (keypressed == '*') Getpass(); } if (a == 4) { // пароль правильний noTone(Play); a = 0; detection = false; } } void Getpass() { // функція введення пароля tone(Play, 2000); for (short i = 0; i < 4; i++) { char keypressed = myKeypad.waitForKey(); if (keypressed == code[i]) a++; } }
Якщо лазер і давач не вирівняні — сигналізація спрацює одразу при завантаженні. Надайте 2 секунди delay(2000) у setup() для вирівнювання.

Project 24-3 · MAX3010x + OLED
Персональний пульсоксиметр

Медичний прилад-монітор: датчик GY-MAX3010x вимірює пульс (BPM) та насичення крові киснем (SpO₂) методом фотоплетизмографії. Дані відображаються на OLED 128×64 з анімованою іконкою серця.

Необхідне обладнання
Arduino UNO
OLED 0.96" I2C 128×64
Датчик GY-MAX3010x
Макетна плата
З'єднувальні дроти
I²C топологія така ж як у проєкті 24-1 — обидва пристрої паралельно на A4/A5.
GY-MAX3010x підключається від 5В (має вбудований стабілізатор 3.3В для власної логіки).
OLED SSD1306 0x3C    MAX3010x 0x57

  • 1
    Зберіть схему: SDA/SCL обох пристроїв до A4/A5. MAX3010x живите від 5В, OLED — за специфікацією модуля.
  • 2
    Встановіть бібліотеки: Adafruit_SSD1306 Adafruit_GFX SparkFun MAX3010x (або MAX30105) через менеджер.
  • 3
    Введіть Лістинг 24-3. Скомпілюйте — при помилках перевірте правильність назв бібліотек.
  • 4
    Завантажте. Прикладіть палець до датчика. На OLED мають з'явитися значення BPM та SpO₂. При відсутності пальця — напис «PLACE FINGER».
Лістинг 24-3 · Пульсоксиметр MAX3010x + OLED
#include <Wire.h> #include <Adafruit_GFX.h> #include <Adafruit_SSD1306.h> #include "MAX30105.h" #include "heartRate.h" #define SCREEN_WIDTH 128 #define SCREEN_HEIGHT 64 Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1); MAX30105 particleSensor; const byte RATE_SIZE = 4; // середнє за 4-ма ударами byte rates[RATE_SIZE]; byte rateSpot = 0; long lastBeat = 0; float beatsPerMinute; int beatAvg; void setup() { Serial.begin(9600); display.begin(SSD1306_SWITCHCAPVCC, 0x3C); display.clearDisplay(); display.setTextColor(WHITE); particleSensor.begin(Wire, I2C_SPEED_FAST); particleSensor.setup(); particleSensor.setPulseAmplitudeRed(0x0A); // менша яскравість LED particleSensor.setPulseAmplitudeIR(0x0A); } void loop() { long irValue = particleSensor.getIR(); if (irValue > 50000) { // палець прикладено display.clearDisplay(); display.setTextSize(1); display.setCursor(0, 0); display.print(" PULSE OXIMETER "); display.drawFastHLine(0, 10, 128, WHITE); if (checkForBeat(irValue)) { // виявлено удар long delta = millis() - lastBeat; lastBeat = millis(); beatsPerMinute = 60 / (delta / 1000.0); if (beatsPerMinute < 255 && beatsPerMinute > 20) { rates[rateSpot++] = (byte)beatsPerMinute; rateSpot %= RATE_SIZE; beatAvg = 0; for (byte x = 0; x < RATE_SIZE; x++) beatAvg += rates[x]; beatAvg /= RATE_SIZE; } } // ── Вивід BPM ──────────────────────────────── display.setTextSize(2); display.setCursor(0, 16); display.print(beatAvg); display.setTextSize(1); display.print(" bpm"); // ── Вивід SpO₂ ─────────────────────────────── display.setCursor(0, 42); display.print("SpO2: "); display.setTextSize(2); display.print(particleSensor.getOxygen(), 1); display.print(" %"); // ── Анімована іконка серця ─────────────────── int pulseLinePos = (int)(irValue - 50000) / 200; display.drawFastVLine(115, 63-pulseLinePos, pulseLinePos, WHITE); if (pulseLinePos > 10) display.drawCircle(115, 16, 8, WHITE); display.display(); } else { // ── Палець не виявлено ──────────────────────── display.clearDisplay(); display.setTextSize(2); display.setCursor(0, 16); display.print(" PLACE "); display.setCursor(0, 36); display.print("FINGER"); display.display(); } }
Принцип PPG: датчик світить червоним (660 нм) та ІЧ (880 нм) через шкіру. Оксигемоглобін поглинає ІЧ більше, ніж дезоксигемоглобін — різниця у відображеному світлі дає SpO₂. Пульсуючий компонент сигналу відповідає ударам серця.
Реалізуйте власний алгоритм розрахунку SpO₂ через різницю поглинання червоного та ІЧ каналів. Додайте buzzer для звукового «бік» на кожен удар серця.
