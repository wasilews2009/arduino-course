01 · ТЕОРІЯ

Що таке матричний індикатор

Принцип побудови

Матричний індикатор — це сітка з 64 світлодіодів (8×8). Кожен піксель задається перетином рядка (R) і стовпця (C). Роздільна здатність — кількість точок по горизонталі та вертикалі.

З'єднання всередині

Усі 64 LED не мають окремих виводів. Катоди об'єднані по 8 у стовпцях, аноди — по рядках. Так 64 контакти скорочуються до 16 виводів.

Проблема «привидів»

Якщо одночасно подати сигнал на R6 та C3, а також R3 та C7 — засвітяться всі 4 перетини! Вирішення — динамічна індикація.

Персистентність зору

Людське ока не помічає мигання швидше за 50–60 Гц. Тому рядки перемикаються настільки швидко, що ми бачимо цілісне зображення.

🔌 Внутрішня схема 8×8 матриці — як це влаштовано ▼

Нумерація виводів матриці починається з нижнього лівого кута, але ніяк не відповідає логіці рядків/стовпців C і R. Це важливо при підключенні!

Для запалення пікселя R6C3 потрібно: HIGH → R6, LOW (земля) → C3.
Другий піксель R3C7: HIGH → R3, LOW → C7.

Якщо увімкнути обидва одночасно, через перехрестя струм потече й через R6C7 та R3C3 — це «привиди». Тому матриця перемикається рядково через мікросекунди.

⚡ Зсувні регістри 74HC595 — навіщо вони тут ▼

Без регістрів Arduino потребував би 16 цифрових виводів. З двома 74HC595 достатньо 3 виводів:

  • data_pin (DS) — послідовна передача даних
  • sh_pin (SH_CP / SRCLK) — тактовий імпульс
  • st_pin (ST_CP / RCLK) — засувка (latch) — оновлює виходи

Перший регістр керує стовпцями (через резистори обмеження струму).
Другий регістр керує рядками (через ULN2003 або напряму).

⚠ Важливо: резистори мають бути на лініях стовпців (1-й регістр), а не рядків. Інакше яскравість LED відрізнятиметься в залежності від кількості активних пікселів.
🎬 Динамічна індикація — алгоритм покроково ▼

Алгоритм виконується у функції loop() із перевіркою таймера через micros():

  1. Збільшити лічильник рядка line від 0 до 7 (циклічно)
  2. Надіслати в 1-й регістр код стовпців для поточного рядка (масив data[])
  3. Надіслати в 2-й регістр код активного рядка — один біт LOW з 8 (інвертований)
  4. Відкрити засувку latch — мікросхема виводить дані на піни
  5. Затримка ~500 мкс → перейти до наступного рядка

Весь цикл із 8 рядків завершується за 4 мс → частота оновлення 250 Гц — без мерехтіння.

📐 digitalWriteFast — чому не звичайний digitalWrite? ▼

Стандартна функція digitalWrite() займає ~3–4 мкс через перевірки та захист. При 64 операціях на кадр це ~256 мкс — помітне мерехтіння.

Прискорена версія через пряму роботу з регістрами порту:

// HIGH PORTD |= _BV(pin); // LOW PORTD &= ~_BV(pin);

Це займає 1–2 тактові цикли (~62–125 нс на 16 МГц) — у 30–60 разів швидше.

🏗 Масштабування матриці: 16×16 і більше ▼

Матриця 8×8 показує 2 цифри або простий символ. Для складніших зображень матриці каскадуються — додаються нові зсувні регістри по вертикалі та горизонталі.

Arduino Uno зі зсувними регістрами впорається з 16×16. Збільшення понад це призводить до помітного мерехтіння через обмежену швидкість shiftOut(). Для більших дисплеїв — SPI або мікросхема MAX7219.

Завдання 1 — Підключення матриці з 74HC595

Компоненти

⬛74HC595 × 2
🟡Резистори 220Ω × 8
🔌Дроти тато-тато × 29
💻Arduino IDE
01

Збірка схеми

Розмістіть на макетній платі два зсувних регістри 74HC595. Перший — для стовпців (з резисторами 220Ω на виходах Q0–Q7), другий — для рядків. Виходи QS (pin 9) першого регістра з'єднайте з DS (pin 14) другого — це каскадний ланцюг.

02

Підключення до Arduino

D2 → DS (pin 14) першого регістра. D3 → SH_CP (pin 11) обох регістрів. D4 → ST_CP (pin 12) обох регістрів. 5V → VCC обох регістрів (pin 16). GND → GND обох регістрів (pin 8).

03

Підключення матриці

Виходи першого регістра (через резистори) → виводи рядків матриці (R1–R8). Виходи другого регістра → виводи стовпців матриці (C1–C8). Увага: нумерація виводів матриці починається з нижнього лівого кута і не відповідає порядку R/C!

04

Завантаження скетча

Відкрийте Arduino IDE, введіть Лістинг 7-6 (нижче), скомпілюйте та завантажте. На матриці має з'явитися слово "Adept" у вигляді прокрутки. Поясніть викладачу роботу кожного блоку коду.

05

Аналіз та модифікація

Спробуйте намалювати власний символ у симуляторі вище, скопіюйте масив data[] та замініть у скетчі. Поекспериментуйте зі значенням to_flick — що відбувається при збільшенні/зменшенні?

Лістинг 7-6 — прокрутка тексту

const int latchPin = 12; // ST_CP зсувного регістра const int clockPin = 8; // SH_CP зсувного регістра const int dataPin = 11; // DS зсувного регістра // Стовпці — бітові маски символів (A, d, e, e, p, t) int data[] = { 0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00, // пусто 0x00,0x3E,0x48,0x88,0x88,0x48,0x3E,0x00, // A 0x00,0x00,0x0C,0x12,0x12,0xFE,0x00,0x00, // d 0x00,0x00,0x7C,0x92,0x92,0x92,0x70,0x00, // e 0x00,0x00,0x7C,0x92,0x92,0x92,0x70,0x00, // e 0x00,0x00,0xFE,0x90,0x90,0x60,0x00,0x00, // p 0x00,0x00,0x10,0x10,0x7E,0x12,0x10,0x00, // t 0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00 // пусто }; // Рядки — один активний рядок на кроці unsigned char tab[] = {0x7f,0xbf,0xdf,0xef,0xf7,0xfb,0xfd,0xfe}; void setup() { pinMode(latchPin, OUTPUT); pinMode(clockPin, OUTPUT); pinMode(dataPin, OUTPUT); } void loop() { for(int n = 0; n < 56; n++) { // зміщення (прокрутка) for(int t = 0; t < 100; t++) { // повторень на позицію for(int num = n; num < 8+n; num++) { // 8 рядків кадру shiftOut(dataPin, clockPin, MSBFIRST, data[num]); // стовпці shiftOut(dataPin, clockPin, MSBFIRST, tab[num-n]); // рядки digitalWrite(latchPin, HIGH); digitalWrite(latchPin, LOW); } } } }

Завдання 5* — Матриця на MAX7219

Спрощене підключення

Замість двох зсувних регістрів і 29 дротів — лише 5 дротів! Модуль MAX7219 сам керує мультиплексуванням, яскравістю та зберіганням кадру.

01

Встановлення бібліотек

В Arduino IDE: Sketch → Include Library → Manage Libraries. Встановіть Adafruit GFX Library та Max72xxPanel. Після встановлення перезапустіть IDE.

02

Підключення модуля

VCC → 5V, GND → GND, DIN → D11 (MOSI), CLK → D13 (SCK), CS → D9. Всього 5 дротів!

03

Завантаження скетча та тестування

Введіть Лістинг 7-7 (нижче). Метод drawPixel() вмикає/вимикає конкретний піксель, write() передає кадр на матрицю. Поекспериментуйте з координатами та яскравістю.

#include <SPI.h> #include <Adafruit_GFX.h> #include <Max72xxPanel.h> int pinCS = 9; int numberOfHorizontalDisplays = 1; // к-сть матриць по горизонталі int numberOfVerticalDisplays = 1; // к-сть матриць по вертикалі Max72xxPanel matrix = Max72xxPanel(pinCS, numberOfHorizontalDisplays, numberOfVerticalDisplays); void setup() { matrix.setIntensity(4); // яскравість 0–15 } void loop() { matrix.drawPixel(3, 4, HIGH); // запалити піксель {3, 4} matrix.write(); // вивести кадр на матрицю delay(300); matrix.drawPixel(3, 4, LOW); // погасити matrix.write(); delay(300); }

Коли каскадувати?

З 74HC595: Uno тягне до 16×16 без мерехтіння. MAX7219: модулі легко з'єднуються в ланцюжок через DIN→DOUT, адресуються за індексом. Для великих екранів — SPI.

06 · КОНТРОЛЬ ЗНАНЬ

Тест — 10 питань

1. Яка роздільна здатність найпоширенішого матричного індикатора?
A4×4 точки
B8×8 точок
C16×16 точок
D32×32 точки
Найпоширеніший матричний індикатор має роздільну здатність 8×8 — 64 пікселі, що ідеально підходить для відображення символів і простих зображень.
2. Скільки виводів має стандартний 8×8 матричний індикатор назовні?
A64 виводи
B8 виводів
C16 виводів
D32 виводи
Завдяки матричному з'єднанню (8 рядків + 8 стовпців) 64 світлодіоди мають лише 16 виводів, що значно спрощує підключення.
3. Чому при одночасному запаленні двох пікселів виникає ефект «привидів»?
AЧерез завелику напругу живлення
BСтрум тече через усі перетини активних рядків і стовпців
CЧерез відсутність резисторів
DЧерез помилку в програмі
Якщо активні рядки R6 і R3 та стовпці C3 і C7, струм тече через усі 4 перетини (R6C3, R6C7, R3C3, R3C7) — засвічуються «зайві» пікселі.
4. Що таке «персистентність зору» у контексті матриць?
AЗдатність матриці зберігати зображення без живлення
BМетод стиснення даних для зображень
CЗдатність ока сприймати швидко змінювані кадри як одне ціле
DВбудована пам'ять мікросхеми 74HC595
Персистентність зору — фізіологічна властивість зорової системи сприймати зображення, що змінюються частіше 50–60 разів на секунду, як одне статичне.
5. Скільки виводів Arduino потрібно для керування матрицею через два 74HC595?
A16
B3
C8
D5
Три виводи: DATA (послідовні дані), CLK (тактовий сигнал), LATCH (засувка). Обидва регістри підключаються паралельно до CLK і LATCH, а DATA каскадується.
6. На яких лініях мають стояти резистори обмеження струму?
AНа лініях рядків (другий регістр)
BНа лініях стовпців (перший регістр)
CНа лініях живлення 5V
DМіж двома регістрами
Резистори на стовпцях (перший регістр) забезпечують рівномірне обмеження струму для кожного LED незалежно від кількості активних пікселів у рядку.
7. Що робить функція latchOn() у програмі?
AВмикає перший рядок матриці
BСкидає всі регістри до нуля
CВідкриває засувку регістра, оновлюючи виходи Q0–Q7
DПочинає передачу даних по SPI
latchOn() подає імпульс HIGH→LOW на ST_CP (RCLK). Це «відкриває» засувку: дані з внутрішнього зсувного регістра передаються на виходи Q0–Q7.
8. Скільки дротів потрібно для підключення модуля MAX7219?
A29
B16
C5
D8
MAX7219 підключається через SPI: VCC, GND, DIN, CLK, CS — лише 5 дротів. Сама мікросхема керує всією індикацією та яскравістю.
9. Яка максимальна розмірність матриці рекомендується для Arduino Uno зі зсувними регістрами?
A8×8
B16×16
C32×32
D64×64
При збільшенні понад 16×16 швидкодія Arduino Uno зі зсувними регістрами недостатня — виникає помітне мерехтіння. Для більших матриць потрібен SPI або MAX7219.
10. Чому у програмі використовується digitalWriteFast замість digitalWrite?
AВона підтримує більше пінів
BВона не потребує підключення бібліотек
CПряма робота з регістрами порту значно швидше — щоб уникнути мерехтіння
DЦе вимога мікросхеми 74HC595
Стандартна digitalWrite() займає ~4 мкс, пряма запис у порт — ~100 нс. При 64 операціях/кадр різниця критична для динамічної індикації без мерехтіння.

Бінарне кодування зображення

Кожен рядок матриці — це один байт (8 біт). Біт 1 = піксель увімкнено, 0 = вимкнено.
Лівий піксель = старший біт (MSB, bit 7). Правий = молодший (LSB, bit 0).
