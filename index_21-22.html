01 · Теоретична частина

Принципи роботи LCD-дисплея

Що таке LCD

LCD (Liquid Crystal Display) — рідкокристалічний дисплей. Складається з двох скляних пластин між якими — матриця комірок, заповнена рідкими кристалами. Молекули рідких кристалів під впливом електричного заряду змінюють поляризацію світла.

Принцип відображення

Поляризоване світло обертається рідким кристалом на 90° і пропускається фільтром. При подачі напруги — обертання не відбувається, фільтр блокує світло: ділянка стає темною. Так формується зображення.

Контролер HD44780

LCD 1602 містить контролер Hitachi HD44780 з паралельним інтерфейсом. Символьний дисплей із підсвічуванням: 2 рядки × 16 символів. Кожен символ — матриця 5×8 пікселів.

4-бітний режим

Дані можна передавати через 4 або 8 виводів. 4-бітний режим використовується для економії виводів мікроконтролера. Підключаються лише D4–D7 дисплея.

⚗️ Як рідкі кристали формують зображення — деталі ▼

Керування LCD здійснюється змінним сигналом. Якщо довго прикладати постійну напругу до однієї комірки — вона вийде з ладу. Тому генеруються спеціальні сигнали змінної форми, щоб до кожної комірки прикладалась змінна напруга.

Матриця LCD має виводи COMx (рядкові) та SEGx (стовпцеві). Комбінуючи сигнали на цих виводах, контролер формує зображення без перегріву комірок.

Саме для цього у LCD 1602 встановлений окремий контролер HD44780 — він приймає команди від мікроконтролера та сам формує необхідні сигнали для кожної комірки.

🔄 Різниця між 4-бітним та 8-бітним режимами ▼

8-бітний режим: дані передаються одразу через 8 ліній (D0–D7). Швидше, але потребує більше виводів Arduino (загалом ~12 виводів).

4-бітний режим: дані передаються у два прийоми через 4 лінії (D4–D7). Трохи повільніше, але достатньо для текстових дисплеїв. Потребує ~6 виводів Arduino.

I²C-адаптер (PCF8574) зменшує кількість виводів до 2 (SDA + SCL). Адреса за замовчуванням: 0x27 або 0x3F. Ідеально для проєктів де задіяно багато інших модулів.

🔤 Кирилиця на LCD 1602 — як це реалізувати ▼

Стандартні бібліотеки для LCD 1602 не підтримують кирилицю. Для відображення тексту українською мовою використовуються зовнішні бібліотеки.

Рекомендована бібліотека: LCD_1602_RUS_ALL — підтримує 16-контактний дисплей та I²C.

  1. Завантажте ZIP-архів бібліотеки
  2. Arduino IDE → Sketch → Include Library → Add .ZIP Library...
  3. Виберіть завантажений архів
  4. Використовуйте приклади з меню для виведення кириличного тексту
✍️ Створення символів користувача — до 8 символів ▼

Бібліотека LiquidCrystal підтримує до 8 символів користувача (індекси 0–7). Кожен символ задається масивом із 8 байт: кожен байт — рядок матриці 5×8.

Алгоритм:

  1. Оголошуємо масив: byte MyChar[8] = { ... };
  2. У setup(): lcd.createChar(0, MyChar); — реєструємо символ
  3. У loop(): lcd.write(byte(0)); — виводимо символ

Для зручного створення бітових масивів символів використовуйте інтерактивний редактор символів у розділі «Символи» цього посібника!

#include <Keypad.h> #include <LiquidCrystal_I2C.h> LiquidCrystal_I2C lcd(0x27, 16, 2); // адреса, 16 стовп., 2 рядки const byte ROWS = 4; // кількість рядків клавіатури const byte COLS = 3; // кількість стовпців клавіатури char hexaKeys[ROWS][COLS] = { {'1','4','7','*'}, {'2','5','8','0'}, {'3','6','9','#'} }; byte rowPins[ROWS] = {5, 4, 3, 2}; // піни рядків клавіатури byte colPins[COLS] = {9, 8, 7, 6}; // піни стовпців клавіатури Keypad customKeypad = Keypad(makeKeymap(hexaKeys), rowPins, colPins, ROWS, COLS); void setup() { Serial.begin(9600); lcd.begin(); // ініціалізація дисплею lcd.backlight(); // вмикаємо підсвічування } void loop() { char customKey = customKeypad.getKey(); if (customKey) { Serial.println(customKey); lcd.setCursor(1, 4); // позиціонування курсору lcd.print(customKey); } }

06 · Контрольні питання

Тест — 10 питань

1. Що означає абревіатура LCD?
ALight Color Display
BLiquid Crystal Display
CLinear Control Device
DLight Crystal Diode
LCD (Liquid Crystal Display) — рідкокристалічний дисплей. Принцип роботи: рідкі кристали змінюють поляризацію світла під дією електричного поля, керуючи пропусканням через поляризаційні фільтри.
2. Скільки символів та рядків відображає LCD 1602?
A8 символів, 1 рядок
B20 символів, 4 рядки
C16 символів, 2 рядки
D32 символи, 2 рядки
LCD 1602: 16 символів × 2 рядки = 32 символи загалом. Кожен символ відображається у матриці 5×8 пікселів. Назва дисплея «1602» якраз і означає 16 стовпців, 2 рядки.
3. Скільки виводів Arduino потрібно для 4-бітного режиму LCD 1602?
A2 виводи (SDA, SCL)
B8 виводів
C6 виводів (RS, E, D4–D7)
D16 виводів
У 4-бітному режимі підключаються: RS, E, D4, D5, D6, D7 — всього 6 виводів Arduino. D0–D3 не використовуються. Для I²C режиму достатньо лише 2 виводів (SDA + SCL).
4. Навіщо вивід R/W LCD 1602 зазвичай підключають до GND?
AДля економії енергії
BДля роботи в режимі запису (Write mode)
CДля захисту від перегріву
DДля збільшення контрастності
R/W = 0 (GND) означає режим запису — передача команд/даних від Arduino до дисплея. R/W = 1 (HIGH) — режим читання зі стану дисплея. Оскільки дисплей використовується як пристрій виводу, R/W завжди тримають LOW.
5. Яка функція встановлює позицію курсора на LCD?
Alcd.moveCursor(col, row)
Blcd.position(col, row)
Clcd.setCursor(col, row)
Dlcd.goto(col, row)
lcd.setCursor(col, row) — перший параметр номер стовпця (0–15), другий — номер рядка (0 або 1). Наприклад, setCursor(2, 1) встановлює курсор у 3-тю позицію другого рядка.
6. Яку максимальну кількість символів користувача підтримує бібліотека LiquidCrystal?
A4 символи
B8 символів
C16 символів
D32 символи
Бібліотека LiquidCrystal підтримує рівно 8 символів користувача (індекси 0–7). Вони зберігаються у CGRAM контролера HD44780. Кожен символ задається масивом із 8 байт (матриця 5×8 пікселів).
7. Яка I²C адреса за замовчуванням для більшості LCD-адаптерів PCF8574?
A0x3C
B0x27
C0x40
D0x08
Більшість I²C-адаптерів PCF8574 для LCD мають адресу 0x27. Деякі виробники використовують 0x3F. Якщо адреса невідома — завантажте I²C Scanner скетч для автоматичного визначення.
8. Яка функція прокручує весь вміст дисплея ліворуч?
Alcd.shiftLeft()
Blcd.moveLeft()
Clcd.scrollDisplayLeft()
Dlcd.scroll(-1)
lcd.scrollDisplayLeft() зміщує весь вміст дисплея на 1 позицію ліворуч. Парна функція: lcd.scrollDisplayRight(). Кожен виклик — одне зміщення на 1 символ, тому у циклі зі delay() отримується анімація прокрутки.
9. Яку розмірність має матриця одного символу на LCD 1602?
A3×5 пікселів
B5×8 пікселів
C8×8 пікселів
D7×5 пікселів
Кожен символ на LCD 1602 відображається у матриці 5 стовпців × 8 рядків = 40 пікселів. Масив символу користувача — 8 байт, кожен байт описує один рядок (використовуються 5 молодших бітів).
10. Яка функція реєструє символ користувача у контролері LCD?
Alcd.addChar(id, arr)
Blcd.defineChar(id, arr)
Clcd.createChar(id, arr)
Dlcd.setChar(id, arr)
lcd.createChar(id, arr) — перший параметр: номер символу (0–7), другий: масив із 8 байт. Викликається у setup(). Для виведення: lcd.write(byte(id)).
