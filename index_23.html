Теоретичний матеріал

Інтерфейс I²C — основи

I²C (Inter-Integrated Circuit) — послідовний синхронний інтерфейс по двох провідниках, розроблений компанією Philips. Призначений для обміну між контролером та периферійними пристроями на малих відстанях зі швидкістю до 400 кбіт/с.

Фізична шина

// I²C шина · Master + Slave пристрої · підтягувальні резистори
Два проводи замість 7+: звичайний LCD 1602 вимагав до 7 пінів Arduino у 4-бітному режимі. З I²C-адаптером (PCF8574) — лише SDA та SCL, які навіть можуть спільно використовуватись іншими пристроями.

Master / Slave архітектура

  • Master (ведучий) — Arduino. Генерує тактовий сигнал SCL, ініціює передачу, звертається до Slave за адресою.
  • Slave (ведений) — дисплей, датчик тощо. Відповідає лише на запит Master. Ніколи не ініціює передачу самостійно.
  • На шині може бути до 127 Slave-пристроїв одночасно (7-бітна адреса).
  • Підтягувальні резистори 4.7 кОм — обов'язкові для коректної роботи.
LCD 2004A — символьний дисплей

Символьний РК-дисплей 20×4 символів на контролері HD44780. Через I²C-адаптер на мікросхемі PCF8574 підключається лише двома дротами.

🔍 Як знайти I²C адресу (I2C Scanner)+

Якщо дисплей не відповідає — перевірте адресу за допомогою I2C Scanner:

I2C Scanner Sketch
#include <Wire.h> void setup() { Wire.begin(); Serial.begin(9600); Serial.println("I2C Scanner"); } void loop() { for(byte addr = 8; addr < 127; addr++) { Wire.beginTransmission(addr); if(Wire.endTransmission() == 0) { Serial.print("Found at 0x"); Serial.println(addr, HEX); // наприклад: 0x27 або 0x3F } } delay(5000); }
Типові адреси: 0x27 (TI PCF8574) або 0x3F (NXP PCF8574). Якщо Scanner нічого не знаходить — поміняйте місцями SDA/SCL.
⚙ Налаштування контрастності+
На задній стороні I²C-адаптера є синій підстроювач (потенціометр). Після підключення і завантаження коду — крутіть його маленькою викруткою до появи чітких символів. Найпоширеніша помилка: потенціометр у крайньому положенні → екран порожній (або чорні блоки).
Ультразвуковий датчик відстані HC-SR04

HC-SR04 вимірює відстань до перешкоди від 2 см до 4 м із точністю ~1 мм. Принцип: посилає ультразвуковий імпульс 40 кГц і вимірює час до повернення відлуння.

ПараметрЗначення
Живлення5В, 15 мА
Діапазон2 см — 400 см
Частота40 кГц (ультразвук)
Кут огляду15°
Точність~1 мм (із фільтрацією)
TRIG пінOUTPUT — імпульс 10 мкс
ECHO пінINPUT — pulseIn() для вимірювання

Розрахунок відстані

Формула розрахунку відстань (см) = час_луна_мкс / 58.2
або: відстань = duration × 0.0343 / 2
// швидкість звуку ≈ 343 м/с; ділимо на 2 (шлях туди і назад)
Не опитуйте датчик частіше 30 мс! Інакше датчик ловить «відлуння» від попереднього власного імпульсу.
📉 Експоненційний фільтр для стабільних показань+

Сирі дані датчика «стрибають». Простий фільтр усуває шуми:

Exponential filter
float distFilt = 0; void loop() { float dist = getDist(TRIG, ECHO); distFilt += (dist - distFilt) * 0.2; // коефіцієнт 0.2 — 20% нового Serial.println(distFilt); delay(50); }

Project 23-1 · DHT11 + LCD 2004A
Домашній клімат-контроль

Arduino зчитує температуру та вологість з датчика DHT11 і кожні 2 секунди виводить дані на LCD 2004A через I²C. Статичний заголовок і підписи не мигають — оновлюються лише цифри.

// Принципова схема · DHT11 + LCD 2004A I²C
  • 1
    Зберіть схему: LCD SDA→A4, SCL→A5, VCC→5V, GND→GND. DHT11: DATA→D2 через резистор 10 кОм до 5В.
  • 2
    Встановіть бібліотеки: LiquidCrystal_I2C (Frank de Brabander) та DHT sensor library (Adafruit).
  • 3
    Запустіть I2C Scanner. Визначте адресу дисплея: 0x27 або 0x3F. Вкажіть її в коді.
  • 4
    Введіть Лістинг 23-1. Зверніть увагу: статичний текст виводиться в setup(), а числа оновлюються в loop() — це уникає мерехтіння.
  • 5
    Якщо символи не видно — відрегулюйте синій потенціометр на задній стороні I²C модуля.
Лістинг 23-1 · Клімат-контроль DHT11 + LCD
#include <Wire.h> #include <LiquidCrystal_I2C.h> #include <DHT.h> #define DHTPIN 2 // датчик на піні D2 #define DHTTYPE DHT11 // тип: DHT11 або DHT22 DHT dht(DHTPIN, DHTTYPE); LiquidCrystal_I2C lcd(0x27, 20, 4); // адреса, 20 колонок, 4 рядки void setup() { dht.begin(); lcd.init(); // ініціалізація lcd.backlight(); // підсвічування ON // Статичний текст (виводимо один раз) lcd.setCursor(0, 0); lcd.print("=== METEO STATION ==="); lcd.setCursor(0, 1); lcd.print("Temp: "); lcd.setCursor(0, 2); lcd.print("Humidity: "); } void loop() { float h = dht.readHumidity(); float t = dht.readTemperature(); if (isnan(h) || isnan(t)) { lcd.setCursor(10, 1); lcd.print("Error "); return; } // Оновлюємо тільки числа — без мерехтіння! lcd.setCursor(10, 1); lcd.print(t, 1); lcd.write(223); lcd.print("C"); // 223 = символ ° lcd.setCursor(10, 2); lcd.print((int)h); lcd.print("% "); // пробіли очищають старі цифри delay(2000); // DHT11 — мінімум 2 сек між зчитуваннями }
Додайте Buzzer на D9, який подає звуковий сигнал якщо температура перевищить 30°C.
DHT11 вимірює вологість ємнісним сенсором із вологоутримуючим полімером. Зміна вологості → зміна ємності → зміна опору. Точність: ±2°C температура, ±5% вологість.

Project 23-2 · HC-SR04 + LCD 2004A
Безконтактний парктронік

Ультразвуковий далекомір HC-SR04 вимірює відстань до перешкоди. На LCD виводяться сантиметри та динамічний прогрес-бар (20 блоків), що візуально показує наближення.

// Принципова схема · HC-SR04 + LCD 2004A I²C
  • 1
    Підключіть HC-SR04: TRIG→D3 (OUTPUT), ECHO→D2 (INPUT), VCC→5V, GND→GND. LCD I²C як у проєкті 23-1.
  • 2
    Введіть Лістинг 23-2. Зверніть увагу на функцію map(distance, 2, 40, 20, 0) — масштабує відстань у кількість заповнених блоків.
  • 3
    Символ (char)255 — заповнений прямокутник у кодуванні HD44780. Це «піксель» прогрес-бару.
  • 4
    Перевірте: піднесіть руку до датчика — прогрес-бар має збільшуватися, а при відведенні — зменшуватись.
Лістинг 23-2 · Парктронік HC-SR04 + LCD
#include <Wire.h> #include <LiquidCrystal_I2C.h> #define TRIG_PIN 3 #define ECHO_PIN 2 LiquidCrystal_I2C lcd(0x27, 20, 4); void setup() { pinMode(TRIG_PIN, OUTPUT); pinMode(ECHO_PIN, INPUT); lcd.init(); lcd.backlight(); lcd.setCursor(0, 0); lcd.print("=== RADAR SYSTEM ==="); } void loop() { // Генерація імпульсу TRIG (10 мкс) digitalWrite(TRIG_PIN, LOW); delayMicroseconds(2); digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10); digitalWrite(TRIG_PIN, LOW); // Вимірювання часу луни long duration = pulseIn(ECHO_PIN, HIGH); // Розрахунок відстані: v_звуку = 343 м/с → /2 (туди і назад) int distance = duration * 0.0343 / 2; if (distance > 40) distance = 40; if (distance < 2) distance = 2; // Вивід дистанції lcd.setCursor(0, 1); lcd.print("Distance: "); lcd.print(distance); lcd.print(" cm "); // Прогрес-бар на 4-му рядку (20 символів) lcd.setCursor(0, 3); int numBars = map(distance, 2, 40, 20, 0); // ближче = більше блоків for (int i = 0; i < 20; i++) { if (i < numBars) lcd.print((char)255); // ■ заповнений блок else lcd.print(" "); } delay(200); // швидке оновлення, але > 30 мс для HC-SR04 }
Додайте LED на D9, який блимає частіше при наближенні до перешкоди.
Кажани та дельфіни використовують ехолокацію на тому самому принципі! HC-SR04 працює на 40 кГц — вище порогу слуху людини (20 кГц).

Project 23-3 · Keypad + LCD 2004A
Розумний сейф із кодовим замком ⭐

Складний проєкт: матрична клавіатура 3×4 + LCD 2004A. Користувач вводить 4-значний пароль. LCD показує статус системи, маскує введення зірочками (***), відображає результат — ACCESS GRANTED або ACCESS DENIED.

Алгоритм: LOCKED → натискання цифр (маскується ★) → # перевіряє → правильно: ACCESS GRANTED (4 сек) → LOCKED. * скидає буфер. Пароль за замовчуванням: 2026
Кнопка KeypadДія
0–9Додати до паролю (макс 4 символи)
*Скинути введений пароль (Clear)

  • 1
    Підключіть Keypad: Рядки R1–R4 → D5,D6,D7,D8. Стовпці C1–C3 → D2,D3,D4. LCD I²C: SDA→A4, SCL→A5.
  • 2
    Встановіть бібліотеку Keypad (Mark Stanley, Alexander Brevig) через менеджер бібліотек.
  • 3
    Введіть Лістинг 23-3. Зверніть увагу на допоміжні функції showLockedScreen(), showAccessGranted(), showAccessDenied() — це гарний стиль модульного коду.
  • 4
    Перевірте: введіть 2026# → ACCESS GRANTED. Введіть неправильний код → ACCESS DENIED. Натисніть * → скидання.
Лістинг 23-3 · Сейф з паролем · Keypad + LCD
#include <Wire.h> #include <LiquidCrystal_I2C.h> #include <Keypad.h> const byte ROWS = 4, COLS = 3; char keys[ROWS][COLS] = { {'1','2','3'},{'4','5','6'}, {'7','8','9'},{'*','0','#'} }; byte rowPins[ROWS] = {5,6,7,8}; byte colPins[COLS] = {2,3,4}; Keypad keypad = Keypad(makeKeymap(keys), rowPins, colPins, ROWS, COLS); LiquidCrystal_I2C lcd(0x27, 20, 4); const String correctPassword = "2026"; // пароль String inputPassword = ""; void setup() { lcd.init(); lcd.backlight(); showLockedScreen(); } void loop() { char key = keypad.getKey(); if (key) { if (key == '*') { inputPassword = ""; showLockedScreen(); } else if (key == '#') { if (inputPassword == correctPassword) showAccessGranted(); else showAccessDenied(); inputPassword = ""; } else if (inputPassword.length() < 4 && key >= '0' && key <= '9') { inputPassword += key; lcd.setCursor(8 + inputPassword.length() - 1, 2); lcd.print("*"); // маскування паролю зірочками } } } void showLockedScreen() { lcd.clear(); lcd.setCursor(0,0); lcd.print("=== SECURITY SYSTEM ==="); lcd.setCursor(0,1); lcd.print("STATUS: LOCKED"); lcd.setCursor(0,2); lcd.print("Code: "); lcd.setCursor(0,3); lcd.print("*:Clear #:Enter"); } void showAccessGranted() { lcd.clear(); lcd.setCursor(0,1); lcd.print(" ACCESS GRANTED "); lcd.setCursor(0,2); lcd.print(" SAFE IS OPEN! "); delay(4000); showLockedScreen(); } void showAccessDenied() { lcd.clear(); lcd.setCursor(0,1); lcd.print(" ACCESS DENIED! "); lcd.setCursor(0,2); lcd.print(" WRONG PASSWORD "); delay(2000); showLockedScreen(); }
