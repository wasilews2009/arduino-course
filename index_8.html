Project 6-1 · Кнопка · tone() · digitalRead()
Ключ азбуки Морзе — кнопка + LED + Buzzer

Класичний телеграфний ключ на Arduino: при натисканні кнопки загоряється LED і звучить buzzer. Поки кнопка натиснута — LED блимає, звук триває. Так вручну можна передавати точки і тире азбуки Морзе.

Лістинг 6-1 · Morse Key (заповніть ***)
const int buzzer = 8; // ← замість *** пін зуммера const int button = 2; // ← замість *** пін кнопки const int led = 9; // ← замість *** пін LED void setup() { pinMode(buzzer, OUTPUT); pinMode(button, INPUT); pinMode(led, OUTPUT); Serial.begin(9600); // ← замість *** } void loop() { int morse = digitalRead(button); Serial.println(morse); if (morse == 1) { digitalWrite(led, HIGH); tone(buzzer, 1000); // ← замість *** } else { digitalWrite(led, LOW); noTone(buzzer); // ← замість *** } delay(50); }
Тире = кнопка натиснута ~300 мс. Точка = ~100 мс. Пауза між символами = 300 мс. Пауза між словами = 700 мс. Тренуйтесь передавати своє ім'я!

Автоматичний передавач азбукою Морзе

Arduino автоматично передає введений текст азбукою Морзе. Рядок символів декодується через switch/case, кожна літера конвертується у послідовність крапок і тирок — одночасно LED + buzzer.

📖 Ключові конструкції Лістингу 6-2+
  • char stringToMorseCode[] — рядок символів для передачі. Змініть на будь-який текст!
  • for(int i=0; i < sizeof(str)-1; i++) — перебирає кожен символ. sizeof()-1 — не передаємо нульовий термінатор \0.
  • toLowerCase(tmpChar) — переводить у нижній регістр для однотипного порівняння в switch.
  • switch(tmpChar) — обирає варіант з тисяч символів ефективніше, ніж ланцюжок if-else.
  • MorseDot() / MorseDash() — окремі функції для крапки та тире. Принцип DRY: Don't Repeat Yourself.
  • LightsOff(delayTime) — пауза між елементами. Час паузи передається як аргумент.
  • dotLen — ключова константа. Всі інші тривалості розраховуються від неї. Змінюйте лише dotLen для прискорення/уповільнення.

Лістинг 6-2 · Автомат азбуки Морзе (завершений)
char stringToMorseCode[] = "Slava Ukraini"; // ваш текст int led12 = 12; // ← замість *** int audio8 = 8; // ← замість *** int note = 1200; // частота тону Морзе (Гц) int dotLen = 100; // крапка = 100 мс int dashLen = dotLen * 3; // тире = 300 мс int elemPause = dotLen; // пауза між елементами int Spaces = dotLen * 3; // пауза між символами int wordPause = dotLen * 7; // пауза між словами void setup() { pinMode(led12, OUTPUT); pinMode(audio8, OUTPUT); Serial.begin(9600); MorseCode(); // запускаємо передачу один раз } void loop() {} // порожній — передача лише в setup() void MorseCode() { for (int i = 0; i < sizeof(stringToMorseCode) - 1; i++) { char tmpChar = toLowerCase(stringToMorseCode[i]); GetChar(tmpChar); delay(1000); // пауза між символами } } void MorseDot() { digitalWrite(led12, HIGH); tone(audio8, note, dotLen); delay(dotLen); } void MorseDash() { digitalWrite(led12, HIGH); tone(audio8, note, dashLen); delay(dashLen); } void LightsOff(int delayTime) { digitalWrite(led12, LOW); noTone(audio8); delay(delayTime); } void GetChar(char tmpChar) { switch (tmpChar) { case 'a': MorseDot(); LightsOff(elemPause); MorseDash(); LightsOff(elemPause); break; // ·− case 'b': MorseDash(); LightsOff(elemPause); MorseDot(); LightsOff(elemPause); MorseDot(); LightsOff(elemPause); MorseDot(); LightsOff(elemPause); break; // −··· case 's': MorseDot(); LightsOff(elemPause); MorseDot(); LightsOff(elemPause); MorseDot(); LightsOff(elemPause); break; // ··· case 'o': MorseDash(); LightsOff(elemPause); MorseDash(); LightsOff(elemPause); MorseDash(); LightsOff(elemPause); break; // −−− case ' ': LightsOff(wordPause); break; // пробіл між словами default: LightsOff(Spaces); // невідомий символ } }
Змініть dotLen = 50 щоб удвічі прискорити передачу. Усі інші тривалості автоматично масштабуються через множники × 3 та × 7.

Ключова інновація — millis() замість delay():
if ((millis() / 300) % 2 == 0) — ділимо мілісекунди на 300, беремо остаток від ділення на 2. Це дає 0 або 1, що чергуються кожні 300 мс → LED блимає. При цьому loop() продовжує виконуватись — давачі опитуються кожні 100 мс, кнопка реагує миттєво.
⚡ Чому millis() краще за delay() для миготіння?+
  • delay() блокує весь контролер — жоден інший код не виконується під час паузи.
  • millis() повертає час з моменту старту в мс. Перевірка (millis()/300)%2 не блокує — виконується миттєво.
  • При delay(1000) для миготіння — натискання кнопки може ігноруватись до закінчення паузи.
  • У цьому проєкті кнопка скидання повинна реагувати завжди — тому millis() обов'язковий.
  • 1
    Живлення: підключіть 5V та GND від Arduino до шин живлення макетної плати (+ та −).
  • 2
    LED: червоний → D8 через 220 Ом → GND. Зелений → D9 через 220 Ом → GND.
  • 3
    Кнопка: один контакт → D2, другий → GND. INPUT_PULLUP — зовнішній резистор не потрібен.
  • 4
    LDR: одна ніжка → 5V, друга → A0. Між другою ніжкою та GND — резистор 10 кОм (дільник напруги).
  • 5
    TMP36: плоска сторона до вас. Ліва ніжка → 5V, права → GND, середня (OUT) → A1.
  • 6
    PIR: VCC → 5V, GND → GND, OUT → D7.
  • 7
    Завантажте скетч. Відкрийте Serial Monitor (9600 бод). Калібруйте пороги: потримайте TMP36 пальцями → червоний LED. Посвітіть ліхтариком на LDR і пройдіть перед PIR → зелений LED миготить. Натисніть кнопку → скидання.
Лістинг 6-3 · Метеостанція-будильник (повний)
const int buttonPin = 2; const int pirPin = 7; const int redLedPin = 8; const int greenLedPin = 9; const int lightPin = A0; const int tempPin = A1; bool alarmActive = false; const float tempThreshold = 27.0; // °C для тривоги const int lightThreshold = 600; // поріг «ранок» void setup() { Serial.begin(9600); pinMode(buttonPin, INPUT_PULLUP); pinMode(pirPin, INPUT); pinMode(redLedPin, OUTPUT); pinMode(greenLedPin, OUTPUT); } void loop() { int lightLevel = analogRead(lightPin); int tempSensorVal = analogRead(tempPin); int motionDetect = digitalRead(pirPin); int buttonState = digitalRead(buttonPin); // LOW=натиснута // TMP36: АЦП → Вольти → °C float voltage = tempSensorVal * 5.0 / 1024.0; float temperatureC = (voltage - 0.5) * 100.0; // 1. ТЕМПЕРАТУРА → червоний LED if (temperatureC > tempThreshold) digitalWrite(redLedPin, HIGH); else digitalWrite(redLedPin, LOW); // 2. БУДИЛЬНИК: СВІТЛО + РУХ → активація if (lightLevel > lightThreshold && motionDetect == HIGH) alarmActive = true; // 3. КНОПКА → скидання if (buttonState == LOW) { alarmActive = false; digitalWrite(greenLedPin, LOW); delay(250); // debounce } // 4. МИГОТІННЯ зеленого через millis() — БЕЗ delay()! if (alarmActive) { if ((millis() / 300) % 2 == 0) digitalWrite(greenLedPin, HIGH); else digitalWrite(greenLedPin, LOW); } else digitalWrite(greenLedPin, LOW); // 5. Serial Monitor для калібрування Serial.print("Температура: "); Serial.print(temperatureC); Serial.print(" °C | Світло: "); Serial.print(lightLevel); Serial.print(" | Рух: "); Serial.println(motionDetect); delay(100); }
Після подачі живлення PIR потребує 30–60 секунд прогріву. Протягом цього часу можуть бути хибні спрацювання будильника.
