Теоретичний матеріал

Що таке Arduino?

Arduino — компанія з відкритим вихідним кодом апаратного та програмного забезпечення, що розробляє та виробляє одноплатні мікроконтролери для створення цифрових пристроїв. Проєкт стартував у 2005 році (автори: Massimo Banzi та David Cuartielles) і відтоді розвивається в геометричній прогресії.

Arduino Uno — флагман лінійки

Платформа на базі мікроконтролера ATmega328, яка стала стандартом для навчання електроніці. Підключається до комп'ютера через USB, дозволяє завантажувати скетчі та взаємодіяти з реальним обладнанням.

Типи пам'яті мікроконтролера
🔷 Flash (ПЗП) — Програмна пам'ять +
Постійний запам'ятовуючий пристрій. Тут зберігається скетч (програмний код). Вміст не змінюється під час роботи — лише при перепрошивці через IDE. Об'єм для Uno: 32 КБ (0.5 КБ займає завантажувач).
🟢 SRAM (ОЗП) — Оперативна пам'ять +
Оперативна пам'ять. Тут живуть змінні — значення яких можуть змінюватися в ході роботи програми. Після перезавантаження МК пам'ять очищається повністю. Об'єм: 2 КБ.
🟡 EEPROM — Енергонезалежна пам'ять користувача +
Electrically Erasable Programmable Read-Only Memory. Значення зберігаються навіть після відключення живлення. Ідеальна для збереження налаштувань, рекордів, калібрувальних даних. Об'єм: 1 КБ. Ресурс: ~100 000 циклів запису.
Типи даних Arduino

Оберіть тип відповідно до діапазону значень — це економить SRAM та прискорює виконання:

Правило: якщо потрібно зберегти число від 0 до 255 — використовуйте byte, не int. Це вдвічі ефективніше по пам'яті.

Оголошення та ініціалізація

Синтаксис
int a = 0, b, c = 10; // кілька змінних через кому byte myVal; // оголошення (значення = 0 для глобальних) float pi = 3.1415; // число з плаваючою комою bool flag = true; // логічний тип char letter = 'A'; // символ ASCII

Два способи оголосити константу

Константи
// Спосіб 1 — ключове слово const (займає SRAM) const byte MY_CONST = 10; // Спосіб 2 — директива #define (зберігається у Flash, не займає SRAM!) #define LED_PIN 13 #define MAX_VALUE 255
Область видимості: глобальна змінна (оголошена поза функцією) доступна з будь-якої частини програми. Локальна змінна (в дужках функції або блоку) існує лише всередині нього.

Практичні завдання

01
Завдання 1 · Project 3-1
Привітаємося зі світЛом — Blink

Класичний "Hello, World!" у світі мікроконтролерів — змусити вбудований світлодіод блимати за допомогою заводського прикладу Blink.

Необхідне обладнання
Arduino UNO
USB-кабель
Комп'ютер з Arduino IDE
  • 1
    Встановіть Arduino IDE на комп'ютер. Запустіть програму та підключіть плату через USB. У меню Інструменти → Плата оберіть Arduino Uno.
  • 2
    Підключіть Arduino до ПК. Перевірте що зелений LED «PWR» горить. Помаранчевий LED (пін 13) повинен блимати — це заводська демо-програма.
  • 3
    Відкрийте приклад: Файл → Приклади → 01.Basics → Blink. Ознайомтеся зі структурою коду.
  • 4
    Натисніть кнопку ✓ Перевірити (Compile). Переконайтесь у повідомленні «Компіляцію завершено». Потім натисніть → Завантажити (Upload). Через 2–3 секунди LED починає блимати.
  • 5
    Проаналізуйте код нижче — зверніть увагу на коментарі (їх кольорове позначення в IDE).
Лістинг 3-1 · Blink
// Функція setup() — виконується ОДИН РАЗ при старті void setup() { // налаштовуємо пін LED_BUILTIN (=13) як ВИХІД pinMode(LED_BUILTIN, OUTPUT); } // Функція loop() — повторюється нескінченно void loop() { digitalWrite(LED_BUILTIN, HIGH); // увімкнути LED (5В на піні) delay(1000); // чекати 1 секунду digitalWrite(LED_BUILTIN, LOW); // вимкнути LED (0В) delay(1000); // чекати 1 секунду }
LED_BUILTIN — це вбудована константа Arduino, що завжди вказує на пін вбудованого світлодіода (для Uno = 13). Дозволяє писати код, сумісний з різними платами.

Блимаючий зовнішній світлодіод

Підключаємо дискретний LED до макетної плати через резистор і програмуємо блимання самостійно.

Необхідне обладнання
Arduino UNO
Макетна плата
Світлодіод (1 шт.)
Резистор 180–220 Ом
Дроти «тато-тато» (2 шт.)
// Принципова схема · LED + резистор
  • 1
    Запустіть Fritzing і відтворіть схему: пін 8 → резистор 180 Ом → анод LED → катод LED → GND.
  • 2
    Зберіть схему на макетній платі. Довша ніжка LED (анод) — до резистора, коротша (катод) — до GND.
  • 3
    Заповніть код нижче: замість ? вкажіть номер піна (8). Додайте delay(1000) після кожного digitalWrite.
  • 4
    Скомпілюйте та завантажте. LED повинен блимати з інтервалом 1 секунда.
Лістинг 3-2 · Ваш код (доповніть ?)
void setup() { pinMode(?, OUTPUT); // вкажіть пін підключення LED } void loop() { digitalWrite(?, HIGH); // увімкнути // ← додайте delay(1000); digitalWrite(?, LOW); // вимкнути // ← додайте delay(1000); }
Не забудьте підключити плату після збирання схеми, щоб уникнути короткого замикання.

Біжучий вогонь — хвиля з 5 світлодіодів

П'ять LED підключаються до пінів 8–12. Вони послідовно блимають, створюючи ефект хвилі вперед і назад.

Необхідне обладнання
Arduino UNO
Макетна плата
5 LED (будь-який колір)
5 резисторів 180 Ом
Дроти «тато-тато»
Алгоритм: LED1 ON → пауза → LED1 OFF → LED2 ON → … → LED5 ON → … → LED2 ON (назад) → повтор.
Лістинг 3.2 · LED Wave
void setup() { pinMode(8, OUTPUT); // LED 1 pinMode(9, OUTPUT); // LED 2 pinMode(10, OUTPUT); // LED 3 pinMode(11, OUTPUT); // LED 4 pinMode(12, OUTPUT); // LED 5 } void loop() { // Хвиля вперед digitalWrite(8, HIGH); delay(500); digitalWrite(8, LOW); digitalWrite(9, HIGH); delay(500); digitalWrite(9, LOW); digitalWrite(10, HIGH); delay(500); digitalWrite(10, LOW); digitalWrite(11, HIGH); delay(500); digitalWrite(11, LOW); digitalWrite(12, HIGH); delay(500); digitalWrite(12, LOW); // Хвиля назад digitalWrite(11, HIGH); delay(500); digitalWrite(11, LOW); digitalWrite(10, HIGH); delay(500); digitalWrite(10, LOW); digitalWrite(9, HIGH); delay(500); digitalWrite(9, LOW); }

Кнопка керує світлодіодом (Digital Input)

Необхідне обладнання
Arduino UNO
Макетна плата

2 резистори 220 Ом
1 кнопка (tact switch)
6 дротів
// Схема · кнопка + LED
Лістинг 3.3 · Button → LED
const int led = 8; // LED на піні 8 const int button = 9; // Кнопка на піні 9 void setup() { pinMode(led, OUTPUT); // пін 8 — вихід pinMode(button, INPUT); // пін 9 — вхід } void loop() { int reads = digitalRead(button); // зчитуємо стан кнопки digitalWrite(led, reads); // передаємо стан на LED }

Завдання 5 · Project 3-3
Світлофор — три LED, реальний алгоритм

Реалізація алгоритму справжнього світлофора: Червоний → Жовтий → Зелений → блимання Зеленого → Жовтий → повтор з використанням констант #define і змінних.

Необхідне обладнання
Arduino UNO
Макетна плата
3 LED (червон., жовт., зелен.)
3 резистори 220 Ом
7 дротів «тато-тато»
Піни підключення: Червоний LED → пін 8  |  Жовтий LED → пін 10  |  Зелений LED → пін 12
Лістинг 3.3 · Світлофор
#define RED_PIN 8 #define YELLOW_PIN 10 #define GREEN_PIN 12 int red_on = 3000; // 3 с червоний int red_yellow = 1000; // 1 с червон.+жовт. int green_on = 3000; // 3 с зелений int green_blink = 500; // 0.5 с між блиманнями int yellow_on = 1000; // 1 с жовтий void setup() { pinMode(RED_PIN, OUTPUT); pinMode(YELLOW_PIN, OUTPUT); pinMode(GREEN_PIN, OUTPUT); } void loop() { // ─── ЧЕРВОНИЙ ─────────────────────────── digitalWrite(RED_PIN, HIGH); delay(red_on); // ─── ЧЕРВОНИЙ + ЖОВТИЙ (підготовка) ──── digitalWrite(YELLOW_PIN, HIGH); delay(red_yellow); digitalWrite(RED_PIN, LOW); digitalWrite(YELLOW_PIN, LOW); // ─── ЗЕЛЕНИЙ ──────────────────────────── digitalWrite(GREEN_PIN, HIGH); delay(green_on); // ─── ЗЕЛЕНИЙ БЛИМАЄ (3 рази) ──────────── for (int i = 0; i < 3; i++) { digitalWrite(GREEN_PIN, LOW); delay(green_blink); digitalWrite(GREEN_PIN, HIGH); delay(green_blink); } digitalWrite(GREEN_PIN, LOW); // ─── ЖОВТИЙ ───────────────────────────── digitalWrite(YELLOW_PIN, HIGH); delay(yellow_on); digitalWrite(YELLOW_PIN, LOW); }
